Technische Dokumentation: user_admin.py

Version: 1.0 Datum: 2026-04-06 Autor: Ralf Krümmel

1. Überblick

Die Datei user_admin.py dient als zentraler Modul für die Verwaltung von Benutzerkonten, insbesondere für administrative Aufgaben wie das Hashing von Passwörtern, das Management von Benutzerstatus und die Durchführung von Datenmanipulationen (z.B. Deaktivierung oder Löschung). Sie interagiert mit anderen Komponenten wie der Authentifizierung (.auth), der Datenbankverwaltung (.database), der Konfiguration (.config) und dem Workspace-Management (.workspace).

2. Importierte Module

Die Datei importiert folgende Module:

  • **__future__**: Für zukünftige Python-Funktionsweise (z.B. typing Verbesserungen).
  • **hashlib**: Für die Erstellung von kryptografischen Hashes (z.B. für Passwörter).
  • **io**: Für die Arbeit mit interner Speicher (z.B. beim Lesen/Schreiben von Daten).
  • **json**: Für die Serialisierung und Deserialisierung von Daten (z.B. für API-Antworten oder Datenbankeinträge).
  • **shutil**: Für Systemoperationen wie das Kopieren oder Verschieben von Dateien (z.B. beim Archivieren von Daten).
  • **time**: Für Zeitmessungen oder das Management von Zeitstempeln.
  • **zipfile**: Für die Erstellung und Manipulation von ZIP-Archiven.
  • **pathlib.Path**: Für eine objektorientierte und plattformunabhängige Handhabung von Dateipfaden.
  • **.auth**: Enthält Funktionen zur Passwort-Hashing (hash_password).
  • **.config**: Enthält die Serverkonfiguration (ServerConfig).
  • **.database**: Enthält die Schnittstelle zur Datenbankverwaltung (SchoolRepository).
  • **.workspace**: Enthält Klassen und Funktionen für das Management von Arbeitsbereichen (WorkspaceManager, slugify).

3. Konstanten und Globale Einstellungen

Die Datei definiert mehrere wichtige Konstanten, die für die Benutzerverwaltung und die Datenhaltung relevant sind:

Konstante Typ Beschreibung
VALID_USER_ROLES tuple[str, ...] Eine Tupel, die die erlaubten Rollen für Benutzer definiert: ("student", "teacher", "admin").
VALID_USER_STATUSES tuple[str, ...] Eine Tupel, die die erlaubten Status für Benutzer definiert: ("active", "inactive", "suspended").
DELETED_ACTOR_LABEL str Der Label, das verwendet wird, um gelöschte Akteure zu kennzeichnen: "[deleted]".
DEFAULT_CHAT_RETENTION_DAYS int Die Standarddauer (in Tagen), über die Chat-Daten beibehalten werden sollen (Standardwert: 180 Tage).
DEFAULT_AUDIT_RETENTION_DAYS int Die Standarddauer (in Tagen), über die Audit-Daten beibehalten werden sollen (Standardwert: 365 Tage).

4. Klassen und Funktionen

(Hinweis: Die bereitgestellte Code-Stücke enthalten nur Importe und Konstanten. Die eigentlichen Methoden der Klasse, die in diesem Snippet definiert werden, sind nicht sichtbar. Die Dokumentation basiert auf den importierten Abhängigkeiten und den definierten Konstanten.) Importe:

  • from .auth import hash_password: Importiert die Funktion zum Hashing von Passwörtern.
  • from .config import ServerConfig: Importiert die Serverkonfiguration.
  • from .database import SchoolRepository: Importiert die Repository-Klasse zur Datenbankinteraktion.
  • from .workspace import WorkspaceManager, slugify: Importiert Klassen für das Workspace-Management und eine Funktion zum Slugifizieren von Strings.

    • Zusammenfassung der Verantwortlichkeiten (basierend auf Importen):

  1. Authentifizierung: Nutzung von hash_password zur sicheren Speicherung von Passwörtern.
  2. Konfiguration: Zugriff auf ServerConfig für Umgebungsparameter.
  3. Datenpersistenz: Interaktion mit SchoolRepository zur Speicherung und Abfrage von Benutzerdaten.
  4. Workspace-Management: Nutzung von WorkspaceManager und slugify zur Verwaltung von Benutzer- oder Datenstrukturen im Workspace.

5. Sicherheitshinweise

  • Passwortsicherheit: Die Verwendung von hash_password aus dem .auth-Modul stellt sicher, dass Passwörter nicht im Klartext gespeichert werden.
  • Rollen- und Statusprüfung: Die Konstanten VALID_USER_ROLES und VALID_USER_STATUSES deuten darauf hin, dass die Logik, die diese Werte verwendet, eine strikte Zugriffskontrolle (Authorization) auf Basis der Benutzerrolle und des Status implementiert.

6. Anwendungsfälle (Hypothetisch)

Dieses Modul wird primär für folgende administrative Aufgaben verwendet:

  • Benutzererstellung/Aktualisierung: Hashing von Passwörtern vor der Speicherung in der Datenbank.
  • Statusänderung: Ändern des Status eines Benutzers (z.B. von active zu inactive oder suspended).
  • Datenarchivierung/Löschung: Durchführung von Prozessen, die Benutzer als DELETED_ACTOR_LABEL kennzeichnen und möglicherweise Daten in ZIP-Archiven komprimieren (shutil, zipfile).
  • Konfigurationsmanagement: Abrufen von Serverkonfigurationen (ServerConfig).

    • --- Zusammenfassung: user_admin.py ist das Backend-Modul für die Verwaltung des Benutzerlebenszyklus, fokussiert auf Sicherheit (Hashing) und Datenintegrität (Repository-Interaktion).
