Funktionen

1. def material_studio_profile_catalog() -> list[dict[str, str]]:

Beschreibung: Diese Funktion ist verantwortlich für die Erstellung eines Katalogs von Material-Profilen. Sie nimmt eine interne Liste von Daten (MATERIAL_STUDIO_PROFILES) entgegen und transformiert diese in eine Liste von Python-Dictionaries, wobei sichergestellt wird, dass jeder Eintrag ein Dictionary ist. Parameter:

  • Keine: Die Funktion benötigt keine externen Parameter.

    • Rückgabe (Return Value):

  • Typ: list[dict[str, str]] (Eine Liste von Dictionaries, wobei jeder Dictionary-Eintrag Schlüssel-Wert-Paare mit String-Typen enthält.)
  • Inhalt: Eine Liste, die alle definierten Material-Profile in einem strukturierten Dictionary-Format enthält.

    • Details zur Implementierung:

def material_studio_profile_catalog() -> list[dict[str, str]]:
return [dict(item) for item in MATERIAL_STUDIO_PROFILES]

Anmerkungen:

  • Die Implementierung verwendet eine List Comprehension, um effizient die Transformation durchzuführen.
  • Es wird angenommen, dass die globale Variable MATERIAL_STUDIO_PROFILES eine iterable Struktur (z.B. eine Liste von Objekten oder Dictionaries) enthält, die in das gewünschte Format umgewandelt werden soll.
  • Die Rückgabe stellt sicher, dass der resultierende Katalog eine konsistente Struktur (Liste von String-basierten Dictionaries) für nachfolgende Verarbeitungsschritte bereitstellt.

    • ---

Abhängigkeiten und Globale Variablen

  • **MATERIAL_STUDIO_PROFILES:** Eine globale Variable (angenommen), die die Rohdaten enthält, aus denen der Katalog generiert wird. Die Struktur dieser Variable ist entscheidend für die korrekte Ausgabe der Funktion.

Technische Dokumentation: Funktion _material_studio_preset_technical_lines

1. Übersicht

Die Funktion _material_studio_preset_technical_lines dient dazu, eine Liste von technischen Vorgaben (Presets) zu generieren, die für verschiedene Programmiersprachen oder Umgebungen (wie HTML, JavaScript, Node.js, Rust) relevant sind. Diese Vorgaben dienen wahrscheinlich dazu, Entwicklern klare Richtlinien für die Bereitstellung und Ausführung von Material in einem spezifischen Schulserver-Kontext zu geben.

2. Funktion

Name: _material_studio_preset_technical_lines Beschreibung: Generiert eine Liste von technischen Anweisungen, die den Entwicklern mitteilen, wie das Material (z.B. eine Anwendung oder ein Skript) im Schulserver bereitgestellt und ausgeführt werden soll, basierend auf der angegebenen Programmiersprache (language). Signature:

def _material_studio_preset_technical_lines(language: str) -> list[str]:

Parameter:

  • **language (str):** Die Sprache, für die die technischen Vorgaben generiert werden soll (z.B. "html", "javascript", "node", "rust").

    • Rückgabe:

  • **list[str]:** Eine Liste von Strings, die die detaillierten technischen Vorgaben darstellen.

3. Logik und Verhalten

Die Funktion verwendet eine Reihe von if-Bedingungen, um spezifische Anweisungen für bekannte Sprachen zu liefern, bevor sie auf eine allgemeine Standardvorgabe zurückgreift.

3.1. Spezifische Sprachesspezifische Handhabung

Die Funktion implementiert spezifische Regeln für die folgenden Sprachen:

  • **language == "html":**
  • Fokussiert auf eine einzelne Hauptdatei (index.html).
  • Verbot von externen Abhängigkeiten (CDNs, Frameworks, Build-Tools).
  • Betonung der sofort sichtbaren Ergebnisse in der HTML-Vorschau.
  • **language == "javascript":**
  • Fokussiert auf eine einzelne Hauptdatei (main.js).
  • Erlaubt nur console.log() für sichtbare Ausgabe.
  • Verbot von DOM- oder Browser-APIs (da serverseitiges JS ausgeführt wird).
  • Verbot externer Bibliotheken.
  • Empfehlung für Input-Handling über stdin oder einen sicheren Fallback.
  • **language == "node":**
  • Fokussiert auf eine einzelne Hauptdatei (main.js).
  • Verbot von Paketinstallationen (nutzung nur der Node.js-Standardbibliothek).
  • Verbot von DOM- oder Browser-APIs.
  • Empfehlung für Input-Handling über stdin oder argv.
  • Betonung der Ausgabe über console.log().
  • **language == "rust":**
  • Fokussiert auf eine einzelne Hauptdatei (main.rs).
  • Verbot externer Crates oder Cargo-Abhängigkeiten.
  • Betonung der Standardbibliothek und println!-Ausgaben.
  • Hinweis zur Erklärung komplexer Konzepte (Ownership, Borrowing, Result) nur, wenn sie für das Beispiel notwendig sind.

3.2. Allgemeine Fallback-Logik

Wenn die Eingabe keine der spezifischen Sprachen entspricht, wird auf eine allgemeine Regelgruppe zurückgegriffen:

  1. Bestimmung des Hauptfiles: Es wird die Datei (main_file) ermittelt, die für die jeweilige Sprache standardmäßig verwendet wird (über DEFAULT_MAIN_FILES).
  2. Generierung der Standardvorgaben: Es wird eine Liste generiert, die folgende Punkte enthält:
  • Die Hauptdatei ({main_file}) als einzige Hauptdatei.
  • Verwendung der Standardbibliothek der jeweiligen Sprache.
  • Verbot externer Bibliotheken oder Build-Schritte.
  • Empfehlung für Input-Handling über stdin oder einen sicheren EOF-Default.

4. Abhängigkeiten und Kontexte

  • **SUPPORTED_LANGUAGES:** Eine externe Konstante oder Variable, die eine Zuordnung von Sprache (language) zu einer standardisierten Label-Form (label) liefert.
  • **DEFAULT_MAIN_FILES:** Eine externe Konstante oder Variable, die eine Zuordnung von Sprache zu einer Standard-Hauptdatei (main_file) liefert.

5. Verwendung und Anwendungsfälle

Diese Funktion ist primär für die Dokumentationsgenerierung oder die Konfiguration von Entwicklungsumgebungen gedacht. Sie stellt sicher, dass Entwickler, die mit einem bestimmten Technologie-Stack arbeiten, sofort die notwendigen, umgebungsabhängigen Best Practices kennen. Beispielhafte Anwendung: Wenn ein Entwickler die Sprache "node" auswählt, erhält er eine Liste von Anweisungen, die ihn zwingen, nur die Node.js-Standardbibliothek zu verwenden und Ergebnisse über console.log() auszugeben.

6. Sicherheit und Robustheit

  • Robustheit: Die Funktion verwendet .get()-Methoden für die Abfrage von Sprachen und Dateien, was eine gewisse Robustheit gegenüber unerwarteten oder nicht definierten Eingaben bietet.
  • Spezifität: Die strikte Trennung zwischen spezifischen Sprachen und dem allgemeinen Fallback stellt sicher, dass die strengsten Regeln für jede Technologie angewendet werden, während eine minimale Basisrichtlinie für unbekannte Sprachen gewährleistet wird.

Technische Dokumentation: material_studio.py

Übersicht

Diese Datei enthält Funktionen, die für die Verarbeitung und Auflösung von Modulen (wahrscheinlich in einem Kurskontext) innerhalb eines "Material Studio"-Systems zuständig sind. Die Hauptfunktion _material_studio_preset_modules dient dazu, eine Liste von Modul-Objekten basierend auf einer Eingabe zu filtern und zu transformieren.

Funktionen

_material_studio_preset_modules(source: dict[str, Any]) -> list[dict[str, Any]]

Diese Funktion nimmt ein Dictionary (source) als Eingabe entgegen und filtert oder transformiert eine Liste von Modulen, basierend auf den im source enthaltenen Informationen. Parameter:

  • source: dict[str, Any]: Ein Eingabedictionary, das Metadaten über die gewünschten Module enthält.

    • Rückgabe:

  • list[dict[str, Any]]: Eine Liste von Modul-Dictionaries, die nach den Filterregeln des source gefiltert und in ein Dictionary-Format konvertiert wurden.

    • Beschreibung: Die Funktion führt folgende Schritte durch:

  1. Direkte Liste-Verarbeitung: Wenn das source-Dictionary einen Schlüssel "modules" enthält, der eine Liste oder ein Tupel ist, werden die Elemente direkt in eine Liste von Dictionaries konvertiert und zurückgegeben.
  2. Kurs-Kontext-Abfrage: Falls kein "modules"-Schlüssel vorhanden ist, wird versucht, einen Kurskontext über die Funktion get_course() abzurufen, basierend auf source.get("course_id"). Falls get_course() fehlschlägt, wird ein leeres Dictionary ({}) zurückgegeben.
  3. Filterung nach Module-IDs:
  • Es werden die module_ids aus source.get("module_ids") extrahiert. Diese werden bereinigt (String-Konvertierung und Entfernen von Leerzeichen) und in einem Set (allowed_module_ids) gespeichert, um eine schnelle O(1)-Lookup-Zeit zu gewährleisten.
  • Die verfügbaren Module aus dem kurskontext (course.get("modules")) werden iteriert.
  • Jedes verfügbare Modul wird geprüft: Wenn ein module_id aus dem Kurs nicht in der erlaubten Liste (allowed_module_ids) enthalten ist, wird das Modul übersprungen (continue).
  • Die gefilterten und gültigen Module werden in die resolved_modules-Liste hinzugefügt.
  1. Rückgabe: Die finale Liste der aufgelösten Module wird zurückgegeben.

    2. Abhängigkeiten:

  • get_course(...): Eine externe Funktion, die den Kurskontext basierend auf einer Kurs-ID abruft.

    • Beispiel (Konzeptuell): Wenn source eine Liste von Modulen enthält, wird diese Liste zurückgegeben. Wenn nicht, wird der Kurs abgerufen und nur jene Module zurückgegeben, deren IDs in module_ids enthalten sind.

Technische Details und Best Practices

  • Typisierung: Die Funktion nutzt Python-Typisierung (dict[str, Any], list[dict[str, Any]]), was die Lesbarkeit und Wartbarkeit erhöht.
  • Robustheit: Die Verwendung von .get() und isinstance() sorgt für eine gewisse Robustheit gegenüber fehlenden oder unerwarteten Werten in der Eingabe (source).
  • Datenbereinigung: Die IDs werden explizit auf Strings konvertiert und Leerzeichen entfernt (.strip()), um sicherzustellen, dass die Vergleichsoperationen konsistent sind.
  • Immutability: Die Funktion erstellt neue Dictionary-Instanzen (dict(item)), bevor sie sie in die Ergebnisliste hinzufügt, was eine gute Praxis ist, um die ursprünglichen Datenstrukturen nicht unbeabsichtigt zu verändern.

Technische Dokumentation: material_studio_instruction_preset_catalog

1. Übersicht

Die Funktion material_studio_instruction_preset_catalog() dient dazu, eine Liste von Anweisungs-Presets (Präformaten) für Lernmaterialien zu generieren, basierend auf einer definierten Quelle von Lerninhalten (MATERIAL_STUDIO_LANGUAGE_PRESET_SOURCES). Diese Presets sind darauf ausgelegt, sowohl Anweisungen für die Erstellung von Arbeitsblättern als auch für die Erstellung von Beispielcodes zu generieren.

2. Funktion

Name: material_studio_instruction_preset_catalog Beschreibung: Generiert eine Liste von Dictionary-Objekten, die als Vorlagen (Presets) für die Erstellung von Lernmaterialien (Arbeitsblätter und Beispielcodes) dienen. Signatur:

def material_studio_instruction_preset_catalog() -> list[dict[str, Any]]:

Parameter:

  • Keine (Die Funktion verwendet interne globale oder importierte Datenstrukturen wie MATERIAL_STUDIO_LANGUAGE_PRESET_SOURCES, SUPPORTED_LANGUAGES und _material_studio_preset_modules).

    • Rückgabe:

  • list[dict[str, Any]]: Eine Liste von Dictionaries, wobei jedes Dictionary ein generiertes Preset repräsentiert.

3. Funktionsweise (Logik)

Die Funktion iteriert über eine definierte Quelle von Lerninhalten (MATERIAL_STUDIO_LANGUAGE_PRESET_SOURCES) und erstellt für jedes verfügbare Modul zwei spezifische Presets: ein Arbeitsblatt-Preset und ein Beispielcode-Preset.

3.1. Iterationsschritt

  1. Quelle durchlaufen: Die Funktion geht durch jeden Eintrag in MATERIAL_STUDIO_LANGUAGE_PRESET_SOURCES.
  2. Datenextraktion: Für jeden Eintrag werden folgende Informationen extrahiert:
  • **Sprache (language):** Wird extrahiert und in Kleinbuchstaben umgewandelt.
  • **Kurs-ID (course_id):** Wird extrahiert oder als Standardwert (language-grundlagen) gesetzt.
  • **Kurs-Label (course_label):** Wird extrahiert oder als Standardwert (course_id) gesetzt.
  • Module-Daten: Die Module für die aktuelle Quelle werden über die interne Funktion _material_studio_preset_modules(source) abgerufen.

3.2. Generierung des Arbeitsblatt-Presets (profile: "worksheet")

Für jedes identifizierte Modul wird ein Preset für die Erstellung eines Arbeitsblatts erstellt:

  • Key: "{language}-worksheet:{module_id}"
  • Label: Der Titel des Moduls.
  • Summary: Eine Zusammenfassung, die die Absicht des Presets beschreibt (Anweisung für ein Arbeitsblatt).
  • Inhaltserstellung:
  • **objectives:** Die Lernziele des Moduls werden extrahiert und bereinigt.
  • **language:** Die unterstützte Sprache wird ermittelt (Standard: language.title()).
  • **prompt (Arbeitsblatt-Anweisung):** Eine zusammengefasste Anweisung, die folgende Komponenten enthält:
  • Eine Einführung, die den Titel und die Lernziele (maximal 3) hervorhebt.
  • Technische Linien (_material_studio_preset_technical_lines(language)).
  • Eine detaillierte Strukturanweisung für das Arbeitsblatt (Einführung, drei Aufgaben mit steigender Schwierigkeit, Musterlösung mit kommentiertem Code, Reflexionsfrage).

3.3. Generierung des Beispielcode-Presets (profile: "example-code")

Für jedes identifizierte Modul wird ein Preset für die Erstellung eines Beispielcodes erstellt:

  • Key: "{language}-example-code:{module_id}"
  • Label: Der Titel des Moduls.
  • Summary: Eine Zusammenfassung, die die Absicht des Presets beschreibt (Anweisung für ein kompaktes Demo-Skript).
  • Inhaltserstellung:
  • **objectives:** Die Lernziele des Moduls werden ebenfalls verwendet.
  • **language:** Die unterstützte Sprache wird ermittelt.
  • **prompt (Beispielcode-Anweisung):** Eine zusammengefasste Anweisung, die folgende Komponenten enthält:
  • Eine Einführung, die den Titel und die Lernziele hervorhebt.
  • Technische Linien.
  • Eine Strukturanweisung für den Beispielcode (alltagsnahe Problemstellung, kompaktes, direkt ausführbares Beispiel mit sichtbarem Ergebnis, Kommentare, kurze Erklärung zur Zielerreichung, keine Aufgaben oder Prüfungsform).

4. Abhängigkeiten und Schlüsselkomponenten

Die Funktion ist stark abhängig von folgenden externen Komponenten:

  • **MATERIAL_STUDIO_LANGUAGE_PRESET_SOURCES:** Die primäre Datenquelle, die die Iteration steuert.
  • **_material_studio_preset_modules(source):** Eine interne Funktion, die aus einem Quelle-Objekt die spezifischen Module eines Lerninhalts extrahiert.
  • **SUPPORTED_LANGUAGES:** Eine Mapping-Struktur, um die korrekte Sprache für die Anzeige zu bestimmen.
  • **_material_studio_preset_technical_lines(language):** Eine interne Funktion, die spezifische technische oder sprachspezifische Hinweise für die Anweisungen hinzufügt.

5. Nutzung und Anwendung

Diese Funktion wird typischerweise am Anfang eines Anwendungsszenarios aufgerufen, um eine vollständige Sammlung von Anweisungen für das Material-Studio zu generieren. Beispielhafte Anwendung:

# Generiert die gesamte Sammlung von Anweisungen
all_presets = material_studio_instruction_preset_catalog()
# Beispiel: Zugriff auf das erste generierte Arbeitsblatt-Preset
first_worksheet_prompt = all_presets[0]
print(first_worksheet_prompt['prompt'])

6. Design- und Stil-Analyse

  • Struktur: Die Funktion ist modular aufgebaut. Sie trennt die Logik der Datenextraktion von der Logik der Prompt-Generierung.
  • Lesbarkeit: Die Verwendung von f-Strings und expliziten Listen zur Zusammenstellung der prompt-Strings macht die generierten Anweisungen sehr strukturiert und leicht nachzuvollziehen.
  • Robustheit: Es gibt grundlegende Fehlerbehandlungen (z.B. source.get(...) or ""), um auf fehlende Schlüssel oder leere Strings zu reagieren, indem diese auf leere Werte gesetzt werden.
  • Output-Format: Die Rückgabe ist konsistent (eine Liste von Dictionaries), was eine einfache Verarbeitung durch nachfolgende Systeme ermöglicht.

Technische Dokumentation: Python-Funktion resolve_material_studio_instruction_preset

1. Übersicht

Die Funktion resolve_material_studio_instruction_preset dient dazu, ein spezifisches Material-Studio-Instruktions-Preset basierend auf einer Schlüssel-ID (preset_key), einem Profil (profile) und einer Sprache (language) aus einem Katalog abzurufen und zurückzugeben.

2. Funktion

Name: resolve_material_studio_instruction_preset Beschreibung: Diese Funktion durchsucht einen Katalog von Material-Studio-Instruktions-Presets, um ein Preset zu finden, das exakt mit der bereitgestellten Schlüssel-ID, dem Profil und der Sprache übereinstimmt. Signatur:

def resolve_material_studio_instruction_preset(preset_key: str, *, profile: str, language: str) -> dict[str, Any] | None:
# ... Funktionskörper ...

Parameter:

Parameter Typ Pflicht Beschreibung
preset_key str Ja Die eindeutige Schlüssel-ID des gesuchten Presets. Wird zur Identifizierung verwendet.
profile str Nein Das zugehörige Profil. Wird zur Filterung nach dem Profil im Katalog verwendet.
language str Nein Die zugehörige Sprache. Wird zur Filterung nach der Sprache im Katalog verwendet.

Rückgabe:

  • **dict[str, Any] | None**: Wenn ein passendes Preset gefunden wird, wird das gesamte Preset als Python-Dictionary zurückgegeben.
  • **None**: Wenn kein Preset mit der angegebenen preset_key, profile und language gefunden wird.

3. Funktionsweise (Logik)

  1. Input-Bereinigung:
  • Der eingehende preset_key wird auf leere Zeichenketten reduziert und Leerzeichen werden entfernt (.strip()).
  • Der profile und die language werden ebenfalls bereinigt und in Kleinbuchstaben umgewandelt (.lower()), um einen case-insensitive Vergleich zu gewährleisten.
  1. Katalog-Durchsuchung:
  • Die Funktion iteriert über alle Elemente (item) im globalen Katalog material_studio_instruction_preset_catalog().
  1. Filterung:
  • Schritt 1 (Key-Vergleich): Es wird geprüft, ob der key des aktuellen Katalog-Eintrags exakt mit dem bereinigten preset_key übereinstimmt. Wenn nicht, wird der aktuelle Eintrag übersprungen (continue).
  • Schritt 2 (Profile-Vergleich): Es wird geprüft, ob das profile des Katalog-Eintrags (nach Bereinigung und Kleinbuchstaben-Konvertierung) mit dem bereinigten resolved_profile übereinstimmt. Wenn nicht, wird die Suche für diesen Eintrag beendet und die Funktion gibt None zurück.
  • Schritt 3 (Language-Vergleich): Es wird geprüft, ob die language des Katalog-Eintrags (nach Bereinigung und Kleinbuchstaben-Konvertierung) mit der bereinigten resolved_language übereinstimmt. Wenn nicht, wird die Suche für diesen Eintrag beendet und die Funktion gibt None zurück.
  1. Erfolg:
  • Wenn alle drei Bedingungen (Key, Profile, Language) für einen Katalog-Eintrag erfüllt sind, wird das gesamte Dictionary dieses Eintrags zurückgegeben.
  1. Fehlgeschlagen:
  • Wenn die Schleife durchläuft und kein passendes Preset gefunden wird, wird am Ende der Funktion None zurückgegeben.

4. Abhängigkeiten

  • **material_studio_instruction_preset_catalog():** Eine externe Funktion oder ein globaler Katalog, der eine Liste von Preset-Objekten (wahrscheinlich Dictionaries) zurückgibt, die durch die Funktion verwendet werden.

Annahme: Der Katalog enthält Einträge wie: {"key": "preset_A", "profile": "Standard", "language": "en", "settings": {...}} Beispiel 1: Erfolgreiche Suche
