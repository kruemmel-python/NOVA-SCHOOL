Technische Dokumentation: curriculum_catalog.py

Übersicht

Diese Datei enthält Hilfsfunktionen zur Erstellung von Fragenstrukturen für einen Lehrplan- oder Quiz-Katalog. Die Funktionen _single und _multi sind spezialisiert darauf, aus gegebenen Eingabeparametern eine standardisierte JSON- oder Dictionary-Struktur für eine Frage zu generieren.

Funktionen

1. _single(question_id: str, prompt: str, options: list[tuple[str, str]], correct: str, explanation: str, *, points: int = 1) -> dict[str, Any]

Diese Funktion erstellt eine Frage mit einer einzigen korrekten Antwort. Beschreibung: Generiert ein Dictionary, das die Struktur einer einzelnen Multiple-Choice-Frage darstellt. Parameter:

Parameter Typ Beschreibung Pflicht Standardwert
question_id str eindeutige Kennung für die Frage. Ja -
prompt str Der eigentliche Fragestellung oder die Aufgabenstellung. Ja -
options list[tuple[str, str]] Eine Liste von Tupeln, wobei jedes Tupel (option_id, label) enthält. Ja -
correct str Die ID oder der Wert der korrekten Antwort. Ja -
explanation str Eine detaillierte Erklärung zur korrekten Antwort. Ja -
points int Die Anzahl der Punkte, die für diese Frage vergeben werden sollen. Nein 1

Rückgabe: Ein Dictionary (dict[str, Any]) mit der Struktur:

{
"id": "...",
"type": "single",
"prompt": "...",
"options": [
{"id": "opt1_id", "label": "Option 1"},
// ... weitere Optionen
],
"correct": ["korrekte_antwort_id"],
"points": 1,
"explanation": "Erklärung zur korrekten Antwort."
}

Beispiel: Wenn options = [("A1", "Antwort A"), ("B2", "Antwort B")] und correct = "A1" ist, wird die korrekte Antwort als Liste (["A1"]) in das Ergebnis aufgenommen. ---

2. _multi(question_id: str, prompt: str, options: list[tuple[str, str]], correct: list[str], explanation: str, *, points: int = 2) -> dict[str, Any]

Diese Funktion erstellt eine Frage mit mehreren korrekten Antworten. Beschreibung: Generiert ein Dictionary, das die Struktur einer Multiple-Choice-Frage mit mehreren möglichen korrekten Antworten darstellt. Parameter:

Parameter Typ Beschreibung Pflicht Standardwert
question_id str eindeutige Kennung für die Frage. Ja -
prompt str Der eigentliche Fragestellung oder die Aufgabenstellung. Ja -
options list[tuple[str, str]] Eine Liste von Tupeln, wobei jedes Tupel (option_id, label) enthält. Ja -
correct list[str] Eine Liste von IDs oder Werten, die als korrekte Antworten gelten. Ja -
explanation str Eine detaillierte Erklärung zu den korrekten Antworten. Ja -
points int Die Anzahl der Punkte, die für diese Frage vergeben werden sollen. Nein 2

Rückgabe: Ein Dictionary (dict[str, Any]) mit der Struktur:

{
"id": "...",
"type": "multi",
"prompt": "...",
"options": [
{"id": "opt1_id", "label": "Option 1"},
// ... weitere Optionen
],
"correct": ["korrekte_antwort_id_1", "korrekte_antwort_id_2"],
"points": 2,
"explanation": "Erklärung zu den korrekten Antworten."
}

Beispiel: Wenn correct = ["A1", "C3"] ist, wird die korrekte Antwort als Liste (["A1", "C3"]) in das Ergebnis aufgenommen.

Nutzungshinweise

Diese Hilfsfunktionen dienen dazu, die Datenstruktur für die Erstellung eines Fragenkatalogs zu standardisieren. Sie sind primär für die Datenvorbereitung gedacht, bevor die Fragen in eine Datenbank oder eine andere Anwendung integriert werden.

Abhängigkeiten

  • typing: Für die Typisierung der Parameter und Rückgabewerte.

def list_courses() -> list[dict[str, Any]]:
"""
Gibt eine Liste aller Kurse aus dem COURSE_CATALOG zurück.

return [dict(course) for course in COURSE_CATALOG.values()]

def get_course(course_id: str) -> dict[str, Any] | None:
"""
Gibt die Details eines Kurses basierend auf seiner ID zurück.
:param course_id: Die eindeutige ID des gesuchten Kurses.
:return: Das Dictionary mit Kursdetails oder None, falls der Kurs nicht existiert.
"""
course = COURSE_CATALOG.get(course_id)
return dict(course) if course else None

  • **COURSE_CATALOG**: Eine globale Struktur (wahrscheinlich ein dict), die die gesamte Datenbank oder das Verzeichnis aller verfügbaren Kurse speichert.
