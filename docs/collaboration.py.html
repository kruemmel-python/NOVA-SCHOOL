Technische Dokumentation: NotebookCollaborationService

1. Überblick

Die Klasse NotebookCollaborationService ist eine zentrale Komponente für die Verwaltung der Zusammenarbeit und des Zustands (State) von Notizbüchern (Notebooks) innerhalb einer Plattform. Sie ist verantwortlich für das Speichern, Abrufen, Synchronisieren und Verwalten der Präsenz von Nutzern während der Arbeit an einem Projekt. Die Implementierung basiert auf einer persistenten Datenbank (vermutlich PostgreSQL aufgrund der SQL-Syntax) und interagiert mit einem SchoolRepository für Datenzugriff und einem WorkspaceManager für die Speicherung der Notebook-Inhalte.

2. Klasse: NotebookCollaborationService

2.1. Eigenschaften (Attributes)

Name Typ Beschreibung
PRESENCE_TTL_SECONDS int Die Zeitdauer (in Sekunden), über die eine Nutzerpräsenz im notebook_presence-Tabelle aktiv bleibt, bevor sie abgelaufen wird. Standardmäßig auf 25 Sekunden gesetzt.
repository SchoolRepository Schnittstelle zur Datenbank-Interaktion (CRUD-Operationen).
workspace_manager WorkspaceManager Schnittstelle zur Verwaltung der eigentlichen Notebook-Inhalte (Speichern/Laden).

2.3. Methoden

2.3.1. snapshot(self, project: dict[str, Any]) -> dict[str, Any]

Erstellt einen vollständigen Snapshot des aktuellen Zustands eines Projekts.

  • Beschreibung: Ruft den aktuellen Zustand des Projekts ab und fasst ihn zusammen, einschließlich der aktuellen Nutzerpräsenz.
  • Parameter:
  • project: Ein Dictionary, das die Projektinformationen enthält (mindestens project_id).
  • Rückgabe: Ein Dictionary mit revision, cells (der aktuellen Zellstruktur) und presence (die aktuelle Nutzerpräsenz).

2.3.2. heartbeat(self, session: Any, project: dict[str, Any], cursor: dict[str, Any] | None = None) -> list[dict[str, Any]]

Aktualisiert die Präsenz eines Nutzers im System.

  • Beschreibung: Fügt oder aktualisiert den aktuellen Standort (cursor) des Nutzers in der notebook_presence-Tabelle und löscht alle Einträge, die älter als PRESENCE_TTL_SECONDS sind.
  • Parameter:
  • session: Informationen über den aktuellen Benutzer (enthält username, user, role).
  • project: Projektinformationen.
  • cursor (optional): Der aktuelle Cursor-Zustand des Nutzers.
  • Rückgabe: Eine Liste von Dictionaries, die die aktuell aktiven Präsenzen für das Projekt zurückgibt.

2.3.3. sync(self, session: Any, project: dict[str, Any], cells: list[dict[str, Any]], base_revision: int, cursor: dict[str, Any] | None = None) -> dict[str, Any]

Führt die eigentliche Synchronisationslogik durch, um den Notebook-Zustand zu aktualisieren.

  • Beschreibung: Vergleicht den aktuellen Zustand (cells) mit einem Basis-Revision (base_revision), normalisiert die Zellen, mergt Änderungen und speichert den neuen Zustand.
  • Parameter:
  • session: Informationen über den aktuellen Benutzer.
  • project: Projektinformationen.
  • cells: Die Liste der Zellen, die synchronisiert werden sollen.
  • base_revision: Die Revision des Zustands, mit dem die Synchronisation beginnen soll.
  • cursor (optional): Der Cursor-Zustand, der für die Präsenz-Update verwendet werden soll.
  • Rückgabe: Ein Dictionary mit der neuen revision, den aktualisierten cells und der neuen presence.
