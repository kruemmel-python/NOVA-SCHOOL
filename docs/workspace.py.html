Sie beinhaltet Funktionen für das Laden von Konfigurationen, das Parsen von Daten und das Management von Dateisystemoperationen. ---

1. Überblick

workspace.py ist eine Python-Klasse, die die notwendigen Komponenten für die Initialisierung und das Management von Arbeitsumgebungen bereitstellt. Sie importiert Abhängigkeiten aus anderen Modulen (config, templates) und scheint die Logik für das Laden von Projekt- oder Workspace-Definitionen zu enthalten.

2. Importierte Module und Klassen

Modul/Klasse Beschreibung Zweck
json Standard-Python-Modul für die Arbeit mit JSON-Daten. Parsen und Serialisieren von Konfigurations- oder Projektdateien.
re Standard-Python-Modul für reguläre Ausdrücke (Regular Expressions). Mustererkennung, Validierung oder Extraktion von Daten aus Strings.
shutil Modul für High-Level-Dateisystemoperationen (Kopieren, Verschieben, Löschen). Verwaltung von Dateien und Ordnerstrukturen (z.B. beim Kopieren von Templates).
pathlib.Path Modul für objektorientierte Dateisystempfad-Manipulation. Sichere und plattformunabhängige Handhabung von Dateipfaden.
ServerConfig (aus .config) Eine Klasse oder ein Datentyp, der Serverkonfigurationen hält. Speicherung von Server-spezifischen Einstellungen.
PROJECT_TEMPLATES (aus .templates) Eine globale oder importierte Struktur, die Projekt-Templates definiert. Bereitstellung der verfügbaren Vorlagen für neue Workspaces.

Funktionale Hypothese: Die Klasse wird wahrscheinlich Methoden enthalten, die folgende Aufgaben erfüllen:

  1. Konfigurladen: Laden von ServerConfig oder spezifischen Workspace-Konfigurationen.
  2. Template-Verwaltung: Nutzung von PROJECT_TEMPLATES zur Erstellung neuer Workspace-Strukturen.
  3. Dateimanipulation: Verwendung von shutil und pathlib zur Erstellung, Kopie oder Bereinigung von Dateien und Ordnern.
  4. Datenverarbeitung: Nutzung von json und re zur Validierung oder Transformation von Workspace-Definitionen.

  • Regex-Nutzung: Wird verwendet, um spezifische Schlüssel-Wert-Paare aus komplexen Konfigurationsdateien zu extrahieren oder um Dateinamen zu validieren.

Technische Dokumentation: WorkspaceManager Klasse

1. Überblick

Die Klasse WorkspaceManager ist eine zentrale Klasse, die für die Verwaltung des Dateisystems und der Projektstruktur innerhalb eines Workspace verantwortlich ist. Sie kümmert sich um die Erstellung von Verzeichnissen, das Materialisieren von Projektstrukturen aus Templates, das Lesen und Schreiben von Dateien, sowie das Management von Notebook-Dateien. Die Klasse arbeitet eng mit einer ServerConfig-Instanz zusammen, um die Pfade für Benutzer- und Gruppenarbeitsräume zu bestimmen.

2. Klasse: WorkspaceManager

2.1. Konstruktor (__init__)

Der Konstruktor initialisiert den Manager und stellt sicher, dass die notwendigen Hauptverzeichnisse für Benutzer- und Gruppenarbeitsräume existieren. Parameter:

  • config: ServerConfig: Eine Instanz der Konfigurationsklasse, die Pfadinformationen liefert.

    • Aktionen:

  1. Speichert die config als Attribut.
  2. Erstellt das Verzeichnis für Benutzerarbeitsräume (self.config.users_workspace_path) mit allen notwendigen Elternverzeichnissen (parents=True) und ignoriert bestehende Verzeichnisse (exist_ok=True).
  3. Erstellt das Verzeichnis für Gruppenarbeitsräume (self.config.groups_workspace_path) mit allen notwendigen Elternverzeichnissen und ignoriert bestehende Verzeichnisse.

2.2. Methoden

2.2.1. ensure_profile_folder(owner_type: str, owner_key: str) -> Path

Stellt sicher, dass das spezifische Verzeichnis für einen bestimmten Besitzer (Benutzer oder Gruppe) existiert. Parameter:

  • owner_type: str: Der Typ des Besitzers (z.B. "user" oder "group").
  • owner_key: str: Der eindeutige Schlüssel des Besitzers.

    • Rückgabetyp:

  • Path: Der Pfad zum erstellten oder bestehenden Verzeichnis.

    • Beschreibung: Navigiert über owner_root und erstellt das entsprechende Verzeichnis.

2.2.2. owner_root(owner_type: str, owner_key: str) -> Path

Bestimmt den Basis-Root-Pfad basierend auf dem Besitzertyp und dem Schlüssel. Parameter:

  • owner_type: str: Der Typ des Besitzers.
  • owner_key: str: Der Schlüssel des Besitzers.

    • Rückgabetyp:

  • Path: Der absolute Pfad zum Root-Verzeichnis.

    • Aktionen:

  1. Normalisiert den owner_key mithilfe der slugify-Funktion.
  2. Wenn owner_type "user" ist, verwendet self.config.users_workspace_path.
  3. Wenn owner_type "group" ist, verwendet self.config.groups_workspace_path.
  4. Löst einen ValueError aus, falls ein unbekannter owner_type übergeben wird.

2.2.3. project_root(project: dict[str, Any]) -> Path

Bestimmt den Haupt-Root-Pfad für ein spezifisches Projekt. Parameter:

  • project: dict[str, Any]: Ein Dictionary, das Projektinformationen enthält (muss owner_type und owner_key enthalten).

    • Rückgabetyp: *

Funktion: slugify(value: str) -> str

Beschreibung: Diese Funktion nimmt einen eingegebenen String (value), bereinigt ihn und konvertiert ihn in eine "Slug"-Form. Eine Slug ist eine URL-freundliche Zeichenkette, die typischerweise für Dateinamen oder URL-Parameter verwendet wird (kleingeschrieben, nur alphanumerische Zeichen und Bindestriche). Parameter:

  • **value (str):** Der Eingabestring, der bereinigt und in eine Slug umgewandelt werden soll.

    • Rückgabe:

  • **str:** Die generierte Slug-String. Wenn der bereinigte String leer ist, wird der Standardwert "workspace" zurückgegeben.

    • Details und Logik:

  1. Bereinigung und Normalisierung (Lowercase & Stripping):
  • lowered = value.strip().lower(): Der eingegebene String wird zuerst um Leerzeichen und Whitespace an den Rändern entfernt und in Kleinbuchstaben umgewandelt.
  • normalized = re.sub(r"[^a-z0-9]+", "-", lowered): Ein regulärer Ausdruck (re.sub) wird verwendet, um alle Zeichen zu ersetzen, die keine Kleinbuchstaben (a-z), Ziffern (0-9) oder Bindestriche (-) sind, durch einen einzelnen Bindestrich (-) zu ersetzen. Dies dient dazu, Sonderzeichen und Leerzeichen in Bindestriche umzuwandeln.
  • normalized = normalized.strip("-"): Jegliche Bindestriche an den Anfang oder Ende der Zeichenkette werden entfernt.
  1. Fallback-Mechanismus:
  • return normalized or "workspace": Wenn nach der Normalisierung die Zeichenkette leer ist (z. B. wenn der ursprüngliche Input nur aus Leerzeichen oder Sonderzeichen bestand), wird der Standardwert "workspace" zurückgegeben.

    • Beispiele:

Eingabe (value) Prozessschritte Ausgabe (slugify(value))
" Meine Datei! Version 1.0 " 1. lower(): " meine Datei! version 1.0 "
2. strip(): "meine Datei! version 1.0"
3. lower(): "meine Datei! version 1.0"
4. re.sub(...): "meine-datei-version-1-0"
5. strip("-"): "meine-datei-version-1-0"
6. Rückgabe "meine-datei-version-1-0"
"Test" 1. lower(): "test"
2. re.sub(...): "test"
3. strip("-"): "test"
4. Rückgabe "test"
"!!!" 1. lower(): "!!!"
2. re.sub(...): "-" (oder leer, je nach Regex-Interpretation, aber hier wird es zu einem Bindestrich)
3. strip("-"): "" (leer)
4. Fallback "workspace"
**"" (Leerzeichen)** 1. lower(): ""
2. re.sub(...): ""
3. strip("-"): ""
4. Fallback "workspace"

Annahmen:

  • Es wird die re-Bibliothek (Regular Expressions) benötigt.
  • Die Funktion ist robust gegenüber verschiedenen Whitespace-Charaktern (Leerzeichen, Tabs, etc.) durch die Verwendung von .strip() und .lower().

    • Verwendete Bibliotheken:

  • re (Standardbibliothek für reguläre Ausdrücke)
