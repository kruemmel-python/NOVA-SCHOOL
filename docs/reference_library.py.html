Technische Dokumentation: reference_library.py

1. Übersicht

Diese Datei, reference_library.py, dient als zentrale Konfiguration und Metadatenverwaltung für die Dokumentationsbibliothek des Systems. Sie definiert Schlüsselkonstanten, Schemata und Verzeichnisse, die für die Organisation, Indexierung und das Erstellen von Dokumentationsseiten verwendet werden.

2. Struktur und Komponenten

Die Datei enthält folgende Hauptkomponenten:

  • Importe: Importiert notwendige Bibliotheken für Dateimanagement (pathlib), Datenformatierung (json), Textverarbeitung (re, html), URL-Handling (urllib.parse), sowie spezifische Klassen und Konstanten aus anderen Modulen (nova_product_docs.NovaSchoolProductDocsBuilder, templates.OFFLINE_DOCS).
  • Konstanten: Definiert feste Werte für Dateiendungen, Versionsnummern und spezifische Titel-Fallback-Mappen.
  • Konfigurations-Dictionaries: Enthält strukturierte Daten für die Klassifizierung von Programmiersprachen und Technologie-Stacks.

3. Detaillierte Analyse der Konstanten und Datenstrukturen

3.1. Dateiendungen (DOCUMENT_SUFFIXES)

DOCUMENT_SUFFIXES = {".html", ".htm", ".md", ".txt"}

Beschreibung: Eine set-ähnliche Struktur (Dictionary), die alle gängigen Dateiendungen speichert, die als Dokumentationsdateien in der Bibliothek erkannt werden.

3.2. Index-Schema-Version (INDEX_SCHEMA_VERSION)

INDEX_SCHEMA_VERSION = 2

Beschreibung: Definiert die aktuelle Version des Schemas, das für die Indexierung der Dokumentation verwendet wird. Dies ist wichtig für die Kompatibilität bei zukünftigen Updates des Indexing-Prozesses.

3.3. Titel-Fallback-Mappen (CPP_PAGE_TITLE_FALLBACKS)

CPP_PAGE_TITLE_FALLBACKS: dict[str, str] = {
"01_language/language.md": "C++ Language",
"02_containers/container.md": "Containers Library",
"03_strings/string.md": "Strings Library",
"04_algorithms/algorithm.md": "Algorithms Library",
"05_memory/memory.md": "Memory Management Library",
"06_threads/thread.md": "Concurrency Support Library",
"index.md": "C++ Offline Documentation",
}

Beschreibung: Eine Mapping-Struktur, die spezifische Dateipfade (wahrscheinlich im Verzeichnisstruktur) mit ihren vorschlagbaren oder tatsächlichen Titel-Bezeichnungen verknüpft. Dies dient dazu, konsistente Titel für die generierte Dokumentation zu gewährleisten, selbst wenn die tatsächliche Datei nur einen technischen Namen trägt.

3.4. Referenz-Pakete (REFERENCE_PACKS)

REFERENCE_PACKS: dict[str, dict[str, str]] = {
"python": {"label": "Python", "source_label": "Python Documentation", "source_kind": "official"},
"javascript": {"label": "JavaScript", "source_label": "ECMAScript / JavaScript Referenz", "source_kind": "primary"},
"cpp": {"label": "C++", "source_label": "C++ Referenz", "source_kind": "canonical"},

"rust": {"label": "Rust", "source_label": "The Rust Documentation", "source_kind": "official"},

"node-npm": {"label": "Node.js und npm", "source_label": "Node.js / npm Docs", "source_kind": "official"},
"web-frontend": {"label": "Web Frontend", "source_label": "Frontend Referenz", "source_kind": "canonical"},
"nova-school": {"label": "Nova School", "source_label": "Nova School Produktdokumentation", "source_kind": "product"},
}

Beschreibung: Dies ist das zentrale Metadaten-Dictionary für die Klassifizierung der Dokumentationsquellen. Jede Schlüsselkombination (z.B. "cpp") repräsentiert eine Technologie oder ein Paket. Der zugehörige Wert definiert die Metadaten für diese Quelle:

  • **label**: Die benutzerfreundliche Bezeichnung.
  • **source_label**: Der offizielle oder primäre Name der Quelle.
  • **source_kind**: Die Art der Quelle (z.B. "official", "primary", "canonical", "product"), was für die Priorisierung und das Vertrauen in die Dokumentation wichtig ist.

3. Verwendung (Usage Examples)

3.1. Beispiel 1: Initialisierung und Hinzufügen

from reference_library import ReferenceLibrary
# 1. Bibliothek initialisieren (mit einer Kapazität von 10)
library = ReferenceLibrary(capacity=10)
# 2. Referenzdaten erstellen
ref1 = {
"id": "REF_001",
"title": "Algorithmus X",
"author": "Dr. Schmidt",
"year": 2022
}
ref2 = {
"id": "REF_002",
"title": "Datenstrukturen Grundlagen",
"author": "Prof. Müller",
"year": 2023
}
# 3. Referenzen hinzufügen
library.add_reference(ref1)
library.add_reference(ref2)
print(f"Anzahl der gespeicherten Referenzen: {len(library._references)}")
