Technische Dokumentation: Berechtigungsmanagement (permissions.py)

1. Überblick

Diese Datei (permissions.py) definiert die Struktur und die Regeln für die Berechtigungsverwaltung innerhalb der Anwendung. Sie legt fest, welche Aktionen (Key) welche Labels und welche Kategorien zugeordnet sind, welche Rollen welche Berechtigungen erhalten und welche Mindestanforderungen (Floors) für bestimmte Rollen gelten.

2. Konstanten und Datenstrukturen

2.1. PERMISSION_DEFINITIONS

Dies ist eine Tupel von Dictionaries, die die vollständige Liste aller definierten Berechtigungen definieren. Jedes Dictionary enthält drei Schlüssel: key, label und category. Struktur: tuple[dict[str, str], ...] Beispiel-Eintrag:

{"key": "project.create", "label": "Projekte erstellen", "category": "Arbeitsbereich"}

2.2. PERMISSION_KEYS

Eine Tupel, die alle key-Strings aus PERMISSION_DEFINITIONS extrahiert. Diese dienen als eindeutige Identifikatoren für alle Berechtigungen. Struktur: tuple[str, ...]

2.3. _ALL_TRUE

Ein Satz (Set), der alle Berechtigungs-Keys enthält, um schnell zu prüfen, ob eine Berechtigung definiert ist. Struktur: set[str]

2.4. ROLE_DEFAULTS

Ein Dictionary, das die Standardberechtigungen für verschiedene Benutzerrollen definiert. Jede Rolle ist ein Dictionary, das den Berechtigungs-Keys als Schlüssel und einen Boolean-Wert (True/False) als Wert enthält. Struktur: dict[str, dict[str, bool]] Definitionen der Rollen:

  • **"student"**: Definiert die Berechtigungen für Studierende.
  • **"teacher"**: Definiert die Berechtigungen für Lehrkräfte.
  • **"admin"**: Definiert die Berechtigungen für Administratoren.

    • Beispiel (Auszug für "student"):

"student": {
"project.create": True,
"workspace.personal": True,
# ... weitere Berechtigungen
"web.access": False,  # Beispiel für eine verweigerte Berechtigung
"ai.use": True,
# ...
}

Beispiel (Auszug für "admin"):

"admin": dict(_ALL_TRUE)  # Alle definierten Berechtigungen sind True

2.5. ROLE_PERMISSION_FLOORS

Ein Dictionary, das die Mindestanforderungen (Floors) für bestimmte Rollen definiert. Struktur: dict[str, dict[str, bool]] Definitionen der Floors:

  • **"admin"**: Erhält alle Berechtigungen (dict(_ALL_TRUE)).

3. Funktionsweise und Logik

Die Datei dient primär als Konfigurations- und Referenzpunkt für das Berechtigungs-System.

  1. Definition: PERMISSION_DEFINITIONS definiert das gesamte Berechtigungs-Schema.
  2. Referenz: PERMISSION_KEYS und _ALL_TRUE ermöglichen eine schnelle Abfrage der gesamten Berechtigungsmenge.
  3. Rollenzuweisung: ROLE_DEFAULTS speichert die spezifische Berechtigungsmatrix für jede Rolle.
  4. Mindestanforderungen: ROLE_PERMISSION_FLOORS definiert, welche Berechtigungen eine Rolle mindestens besitzen muss, um bestimmte Funktionen nutzen zu können.

    5. Implizite Logik (für das verwendete System): Ein Benutzer erhält eine bestimmte Berechtigung, wenn:

  5. Die Berechtigung in der ROLE_DEFAULTS für die zugewiesene Rolle explizit auf True gesetzt ist.
  6. (Optional, aber impliziert durch die Struktur) Die Berechtigung erfüllt die Anforderungen der ROLE_PERMISSION_FLOORS.

4. Zusammenfassung der Rollen-Berechtigungen

Rolle Schlüsselberechtigungen (Beispiele) Besonderheiten
student project.create, run.*, ai.use, docs.read Begrenzte Zugriffsberechtigungen (z.B. web.access: False).
teacher project.create, web.access, curriculum.manage, teacher.chat.* Erweitertes Management- und Moderationsrecht.
admin Alle Berechtigungen (dict(_ALL_TRUE)) Voller Zugriff auf alle Funktionen.

--- Hinweis: Diese Dokumentation basiert ausschließlich auf dem bereitgestellten Codeausschnitt. Für eine vollständige Funktionsbeschreibung des Berechtigungsmechanismus müssten die Implementierungsdateien (die die Logik zur Überprüfung der Berechtigungen ausführen) konsultiert werden.

Technische Dokumentation: permissions.py

Übersicht

Die Datei permissions.py enthält eine Reihe von Funktionen, die für die Verwaltung, Normalisierung und Auflösung von Berechtigungen (Permissions) in einem System zuständig sind. Diese Funktionen ermöglichen es, Berechtigungen basierend auf Rollen, Gruppenüberschreibungen und Benutzerüberschreibungen zu bestimmen.

Funktionen

1. normalize_permission_overrides(raw: dict[str, Any] | None] -> dict[str, bool]

Zweck: Normalisiert ein rohes Dictionary von Berechtigungsüberschreibungen in ein Dictionary, das nur str-Schlüssel und bool-Werte enthält. Parameter:

  • raw: dict[str, Any] | None: Ein optionales Dictionary, das die Rohdaten der Berechtigungsüberschreibungen enthält.

    • Rückgabe:

  • dict[str, bool]: Ein Dictionary, das die bereinigten Berechtigungen enthält.

    • Beschreibung: Diese Funktion filtert das eingegebene Dictionary, indem sie nur Schlüssel behält, die in PERMISSION_KEYS definiert sind, und ignoriert Werte, die None sind. Alle Werte werden explizit in einen Boolean-Wert umgewandelt (bool(value)). Details:

  • Wenn raw None ist, wird ein leeres Dictionary zurückgegeben.
  • Schlüssel, die nicht in PERMISSION_KEYS enthalten sind, oder Werte, die None sind, werden ignoriert.

    • ---

2. resolve_permissions(role: str, group_overrides: list[dict[str, Any]] | None = None, user_overrides: dict[str, Any] | None = None) -> dict[str, bool]

Zweck: Berechnet das effektive Set von Berechtigungen für eine bestimmte Rolle unter Berücksichtigung von Gruppenüberschreibungen und Benutzerüberschreibungen. Parameter:

  • role: str: Die Rolle, für die die Berechtigungen aufgelöst werden sollen.
  • group_overrides: list[dict[str, Any]] | None: Eine Liste von Dictionaries, die spezifische Berechtigungen für Gruppenüberschreibungen enthalten.
  • user_overrides: dict[str, Any] | None: Ein optionales Dictionary mit Berechtigungsüberschreibungen, die durch den Benutzer gesetzt werden sollen.

    • Rückgabe:

  • dict[str, bool]: Das finale Dictionary mit den effektiven Berechtigungen (Schlüssel: Berechtigungsname, Wert: True oder False).

    • Beschreibung: Diese Funktion folgt einer hierarchischen Logik zur Bestimmung der Berechtigungen:

  1. Basiswert: Sie beginnt mit den Standardberechtigungen (ROLE_DEFAULTS) der angegebenen role (Standardwert ist "student").
  2. Gruppenüberschreibungen: Sie prüft die group_overrides.
  • Wenn für eine bestimmte Berechtigung in allen Gruppenüberschreibungen ein False gesetzt ist, wird die Berechtigung auf False gesetzt.
  • Wenn für eine bestimmte Berechtigung in mindestens einem Gruppenüberschreibungen ein True gesetzt ist, wird die Berechtigung auf True gesetzt.
  1. Benutzerüberschreibungen: Die Ergebnisse der Normalisierung von user_overrides werden verwendet, um die Berechtigungen zu überschreiben.
  2. Rollen-Floor-Regeln: Sie prüft die ROLE_PERMISSION_FLOORS für die Rolle. Wenn ein Wert in ROLE_PERMISSION_FLOORS für einen Schlüssel True ist, wird die entsprechende Berechtigung auf True gesetzt (was eine Art "Floor"-Regel darstellt).

    3. ---

3. permission_catalog() -> list[dict[str, str]]

Zweck: Generiert einen Katalog aller definierten Berechtigungen. Parameter:

  • Keine.

    • Rückgabe:

  • list[dict[str, str]]: Eine Liste von Dictionaries, wobei jedes Dictionary eine Berechtigung definiert (wahrscheinlich Schlüssel: Berechtigungsname, Wert: Beschreibung oder Definition).

    • Beschreibung: Diese Funktion konvertiert die internen PERMISSION_DEFINITIONS in ein Format, das für die Dokumentation oder die Darstellung der Berechtigungen geeignet ist. ---

4. allowed_tool_names(permissions: dict[str, bool]) -> tuple[str, ...]

Zweck: Filtert ein Set von Berechtigungen, um eine Tupel von Berechtigungsnamen zurückzugeben, die für die Ausführung eines bestimmten Tools erlaubt sind. Parameter:

  • permissions: dict[str, bool]: Ein Dictionary der effektiven Berechtigungen (Schlüssel: Berechtigungsname, Wert: True).

    • Rückgabe:

  • tuple[str, ... ]: Ein Tupel von Strings, die die Namen der erlaubten Berechtigungen darstellen.

    • Beschreibung: Diese Funktion wählt alle Schlüssel aus dem permissions-Dictionary aus, für die der Wert True ist, sortiert diese alphabetisch und gibt sie als Tupel zurück. ---

Abhängigkeiten und Konstanten (Impliziert)

Die Funktion permissions.py basiert auf folgenden internen Konstanten, die in der Datei definiert sein müssen:

  • **PERMISSION_KEYS**: Eine Liste oder ein Satz von Strings, die die gültigen Berechtigungsnamen definieren.
  • **ROLE_DEFAULTS**: Ein Dictionary, das Standardberechtigungen für verschiedene Rollen definiert.
  • **PERMISSION_DEFINITIONS**: Eine Liste von Dictionaries, die die detaillierten Definitionen aller Berechtigungen enthalten.
  • **ROLE_PERMISSION_FLOORS**: Ein Dictionary, das spezifische Regeln definiert, die die Berechtigungen für bestimmte Rollen beeinflussen (Floor-Regeln).

Zusammenfassung der Nutzungsszenarien

  1. Berechtigungsprüfung: Rufen Sie resolve_permissions(role, group_overrides, user_overrides) auf, um festzustellen, welche Berechtigungen eine bestimmte Rolle unter gegebenen Bedingungen besitzt.
  2. Tool-Filterung: Rufen Sie allowed_tool_names(effective_permissions) auf, um schnell zu prüfen, welche Tools ein Benutzer aufgrund seiner Berechtigungen nutzen darf.
  3. Katalog-Management: Nutzen Sie permission_catalog() zur Erstellung oder Aktualisierung der Definitionen aller Berechtigungen.
  4. Datenbereinigung: normalize_permission_overrides wird intern verwendet, um externe Daten sauber in das interne Boolean-Format zu überführen.
