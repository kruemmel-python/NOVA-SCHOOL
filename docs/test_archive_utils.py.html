Sie stellt sicher, dass die Funktionen zur sicheren Extraktion von ZIP-Archiven (extract_zip_safely) korrekt und robust funktionieren. Die Datei importiert spezifische Funktionen aus nova_school_server.archive_utils und nutzt Standard-Python-Module (stat, tempfile, unittest, zipfile, pathlib) sowie das unittest-Framework zur Durchführung der Tests.

Technische Dokumentation: test_archive_utils.py

1. Übersicht

Diese Datei enthält eine Reihe von Unit-Tests, die die Funktionalität der Funktion extract_zip_safely (die in einer anderen Datei definiert ist, aber hier getestet wird) validieren. Die Tests konzentrieren sich darauf, wie die Funktion mit verschiedenen Szenarien im ZIP-Archiv umgeht, insbesondere bei der Sicherstellung der Sicherheit beim Extrahieren von Dateien.

2. Zielsetzung der Tests

Das Hauptziel dieser Tests ist es, sicherzustellen, dass die Funktion extract_zip_safely folgende Sicherheitsanforderungen erfüllt:

  1. Sichere Extraktion normaler Dateien: Die Funktion muss Dateien korrekt und sicher in das Zielverzeichnis extrahieren können.
  2. Verweigerung von Pfad-Traversal (Path Traversal): Die Funktion muss verhindern, dass Benutzer durch das Verwenden von relativen Pfaden (z.B. ../../) auf Dateien außerhalb des Zielverzeichnisses zugreifen.
  3. Verweigerung von Symbollinks: Die Funktion muss die Extraktion von Dateien, die als Symbollinks konfiguriert sind, verhindern, um Sicherheitslücken zu vermeiden.

3. Testfälle im Detail

Die Datei enthält drei spezifische Testfälle, die jeweils eine bestimmte Sicherheitsaspekt prüfen:

3.1. Testfall: test_extract_zip_safely_writes_normal_files

Beschreibung: Dieser Test stellt sicher, dass die Funktion korrekt und erfolgreich eine normale Datei aus einem ZIP-Archiv extrahieren kann. Ablauf:

  1. Ein temporäres Verzeichnis wird erstellt.
  2. Ein ZIP-Archiv (safe.zip) wird erstellt, das eine Datei (folder/main.py) enthält.
  3. Die Funktion extract_zip_safely wird auf dieses Archiv angewendet.
  4. Es wird überprüft, ob der Inhalt der extrahierten Datei (main.py) exakt mit dem ursprünglichen Inhalt übereinstimmt.

    5. Erwartetes Ergebnis: Der Inhalt der extrahierten Datei muss exakt dem ursprünglichen Inhalt entsprechen.

3.2. Testfall: test_extract_zip_safely_rejects_traversal

Beschreibung: Dieser Test validiert die Schutzmechanismen gegen Path Traversal. Er prüft, ob die Funktion verhindert, dass Benutzer durch Manipulierung des Pfades (z.B. durch ../../) auf Dateien außerhalb des definierten Zielverzeichnisses zugreifen. Ablauf:

  1. Ein temporäres Verzeichnis wird erstellt.
  2. Ein ZIP-Archiv (traversal.zip) wird erstellt, das einen Pfad enthält, der außerhalb des Zielverzeichnisses liegt (z.B. ../../outside.txt).
  3. Die Funktion extract_zip_safely wird auf dieses Archiv angewendet.
  4. Es wird erwartet, dass die Funktion eine PermissionError (oder eine ähnliche Fehlermeldung, die auf Zugriffsbeschränkungen hinweist) ausgibt, da der Zugriff außerhalb des erlaubten Bereichs blockiert werden muss.

    5. Erwartetes Ergebnis: Es muss eine PermissionError ausgelöst werden, was beweist, dass der Pfad-Traversal erfolgreich abgefangen wurde.

3.3. Testfall: test_extract_zip_safely_rejects_symlink_entries

Beschreibung: Dieser Test prüft die Sicherheit bezüglich Symbollinks. Er stellt sicher, dass die Funktion die Extraktion von Dateien verhindert, die als Symbollinks konfiguriert sind, um das Ausführen von Befehlen oder das Umgehen von Dateisystembeschränkungen zu verhindern. Ablauf:

  1. Ein temporäres Verzeichnis wird erstellt.
  2. Ein ZIP-Archiv (symlink.zip) wird erstellt, das eine spezielle ZipInfo-Struktur enthält, die den Eintrag als Symbollink kennzeichnet (info.create_system = 3 und eine spezifische external_attr für einen Link).
  3. Die Funktion extract_zip_safely wird auf dieses Archiv angewendet.
  4. Es wird erwartet, dass die Funktion eine PermissionError ausgibt, da die Extraktion von Symbollinks blockiert werden muss.

    5. Erwartetes Ergebnis: Es muss eine PermissionError ausgelöst werden, was beweist, dass die Extraktion von Symbollinks erfolgreich verhindert wurde.

4. Voraussetzungen

Um diese Tests ausführen zu können, müssen folgende Bibliotheken installiert sein:

  • unittest (Standardbibliothek)
  • zipfile (Standardbibliothek)
  • pathlib.Path (Standardbibliothek)
  • tempfile (Standardbibliothek)
  • stat (Standardbibliothek)

5. Zusammenfassung

Die Testsuite ArchiveUtilsTests ist ein effektives Werkzeug, um die Robustheit und die Implementierung der Sicherheitsmechanismen in extract_zip_safely zu verifizieren. Sie deckt kritische Angriffsvektoren wie Pfad-Traversal und die Extraktion von Symbollinks ab.
