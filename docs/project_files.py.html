Technische Dokumentation: project_files.py

Übersicht

Diese Dokumentation beschreibt die Funktionen der Python-Datei project_files.py, die Werkzeuge zur Erstellung und Auflistung von Projekt-Snapshots (Klonen von Projektstrukturen) bereitstellt. Die Funktionen verwenden das pathlib-Modul für Pfadmanipulation und das shutil-Modul für Dateikopien.

Abhängigkeiten

  • pathlib: Für die moderne und objektorientierte Pfadmanipulation.
  • shutil: Für das Kopieren von Dateien und Verzeichnissen.

Funktionen

1. copy_project_snapshot(project_root: Path, target_root: Path) -> list[str]

Diese Funktion kopiert die Struktur eines Projektverzeichnisses (project_root) in ein Zielverzeichnis (target_root). Sie durchläuft alle Dateien und Verzeichnisse im Quellprojekt und kopiert sie in das Ziel, wobei bestimmte Verzeichnisse ignoriert werden. Parameter:

  • project_root: Path: Der Pfad zum Quellprojektverzeichnis, das gespiegelt werden soll.
  • target_root: Path: Der Pfad zum Zielverzeichnis, in das die Kopie des Projekts erstellt werden soll.

    • Verhalten:

  1. Zielverzeichnis erstellen: Stellt sicher, dass das target_root-Verzeichnis existiert (erstellt es bei Bedarf mit allen notwendigen Elternverzeichnissen).
  2. Durchlauf: Durchläuft alle Dateien und Verzeichnisse im project_root rekursiv (rglob("*")).
  3. Sortierung: Die Elemente werden nach ihrem kleingeschriebenen, posix-konformen Namen sortiert, um eine konsistente Kopierreihenfolge zu gewährleisten.
  4. Filterung (Ignorieren): Verzeichnisse, die den String ".nova-school" in ihren relativen Pfad enthalten, werden ignoriert (nicht kopiert).
  5. Kopierprozess:
  • Für Dateien wird shutil.copy2() verwendet, um die Datei und ihre Metadaten zu kopieren.
  • Für Verzeichnisse wird das Zielverzeichnis erstellt, falls es noch nicht existiert.
  1. Rückgabe: Gibt eine Liste von Strings zurück, die die relativen Pfade der erfolgreich kopierten Dateien im Zielverzeichnis darstellen (im POSIX-Format).

    2. Beispiel:

# Beispielanwendung (Konzeptuell)
project_dir = Path("./my_project")
target_dir = Path("./backup_snapshot")
copied_list = copy_project_snapshot(project_dir, target_dir)
print(copied_list)
# Ausgabe könnte: ['src/main.py', 'tests/test_utils.py', ...]

2. list_snapshot_files(snapshot_root: Path) -> list[str]

Diese Funktion listet alle Dateien im angegebenen Snapshot-Verzeichnis rekursiv auf und gibt deren relative Pfade zum Snapshot-Root zurück. Parameter:

  • snapshot_root: Path: Der Pfad zum Verzeichnis, das den Snapshot enthält.

    • Verhalten:

  1. Durchlauf: Durchläuft alle Elemente im snapshot_root rekursiv (rglob("*")).
  2. Filterung: Wählt nur Elemente aus, die Dateien (path.is_file()) sind.
  3. Sortierung: Die gefundenen Dateien werden nach ihrem kleingeschriebenen, posix-konformen Namen sortiert.
  4. Rückgabe: Gibt eine Liste von Strings zurück, die die relativen Pfade der gefundenen Dateien zum snapshot_root darstellen (im POSIX-Format).

    5. Beispiel:

# Beispielanwendung (Konzeptuell)
snapshot_dir = Path("./snapshot_data")
file_list = list_snapshot_files(snapshot_dir)
print(file_list)
# Ausgabe könnte: ['config/settings.json', 'data/input_01.csv', ...]

Zusammenfassung der Logik

Die Datei dient als Utility-Layer für das Management von Projekt-Backups oder Snapshots. Sie stellt sicher, dass:

  1. Kopien von Projektstrukturen konsistent und sortiert erstellt werden (copy_project_snapshot).
  2. Die Inhalte eines bereits existierenden Snapshots einfach aufgelistet werden können (list_snapshot_files).
  3. Bestimmte, möglicherweise nicht relevanten Teile des Projekts (z.B. .nova-school-Verzeichnisse) aus dem Backup ausgeschlossen werden.

Technische Dokumentation: read_text_preview Funktion

1. Übersicht

Die Funktion read_text_preview dient dazu, einen Textausschnitt (Preview) aus einem gegebenen Verzeichnisbaum (snapshot_root) zu lesen und zurückzugeben. Sie sucht nach einem bevorzugten Pfad oder durchsucht rekursiv alle Dateien im Verzeichnisbaum, um den Text zu extrahieren und auf eine maximale Zeichenanzahl zu beschränken.

2. Funktion

def read_text_preview(snapshot_root: Path, preferred_path: str | None = None, max_chars: int = 3200) -> dict[str, str]:
# ... (Implementierung)

2.1. Parameter

Parameter Typ Beschreibung Standardwert
snapshot_root Path Der Pfad zum Verzeichnisbaum, aus dem die Suche gestartet wird. Erforderlich
preferred_path `str None` Optionaler spezifischer Pfad innerhalb von snapshot_root, den zuerst gesucht werden soll. Wenn vorhanden, wird dieser als primäre Quelle verwendet. None
max_chars int Die maximale Anzahl von Zeichen, die aus dem gelesenen Inhalt zurückgegeben werden sollen. Standardmäßig auf 3200 gesetzt. 3200

2.2. Rückgabe (Return Value)

Die Funktion gibt ein Dictionary (dict[str, str]) zurück, das entweder den gefundenen Pfad und den entsprechenden Textinhalt enthält oder eine leere Struktur, falls keine Datei gefunden wird. Struktur des zurückgegebenen Dictionaries:

  • "path": Der relative Pfad zur gefundenen Datei innerhalb von snapshot_root, dargestellt als POSIX-Pfad (z.B. /subdir/file.txt).
  • "content": Der lesbare Textinhalt der Datei, begrenzt auf max_chars Zeichen.

    • Fallfall (Keine Datei gefunden): Wenn keine Datei im gesamten Verzeichnisbaum gefunden wird, wird das folgende Dictionary zurückgegeben:

{"path": "", "content": ""}

3. Funktionsweise (Logik)

Die Funktion folgt einer Prioritätsstruktur zur Textextraktion:

  1. **Prüfung des bevorzugten Pfads (preferred_path):**
  • Wenn preferred_path angegeben ist, wird zuerst geprüft, ob ein Dateipfad unter snapshot_root / preferred_path existiert und eine Datei ist.
  • Sollte die Datei existieren, wird ihr Inhalt gelesen, abgeschnitten auf max_chars Zeichen, und das Ergebnis wird sofort zurückgegeben.
  1. Rekursive Durchsuchung des Verzeichnisbaums:
  • Falls preferred_path None ist oder die Datei nicht existiert, wird eine rekursive Suche (snapshot_root.rglob("*")) über alle Dateien im gesamten Verzeichnisbaum durchgeführt.
  • Die Dateien werden nach ihrem alphabetischen (case-insensitive) Pfad sortiert, um eine konsistente Suchreihenfolge zu gewährleisten.
  • Sobald die erste Datei gefunden wird, wird ihr Inhalt gelesen, abgeschnitten auf max_chars Zeichen, und das Ergebnis wird zurückgegeben.
  1. Fehlfall:
  • Wenn die gesamte Suche abgeschlossen ist und keine Datei gefunden wurde, wird das Standard-Fehler-Dictionary zurückgegeben.

4. Abhängigkeiten

  • **pathlib.Path**: Für die Manipulation von Dateipfaden und Verzeichnisstrukturen.
  • **str und dict**: Für die Rückgabe von Datenstrukturen.
  • Dateizugriff: Die Methode read_text() (impliziert durch die Verwendung von Path.read_text()) wird verwendet, um den Inhalt der Dateien zu lesen.

5. Beispielanwendung (Pseudocode)

from pathlib import Path
# Angenommen, snapshot_root ist ein Path-Objekt zu einem Ordner
snapshot_root = Path("/path/to/your/snapshot")
# Beispiel 1: Suche nach einem bevorzugten Pfad
result1 = read_text_preview(
snapshot_root=snapshot_root,
preferred_path="config/settings.txt",
max_chars=100
)
# result1 enthält den Inhalt von config/settings.txt
# Beispiel 2: Allgemeine Suche
result2 = read_text_preview(
snapshot_root=snapshot_root,
max_chars=500
)
# result2 enthält den Inhalt der ersten gefundenen Datei im Baum
