Technische Dokumentation: reference_import_web.py

1. Überblick

Die Python-Skript reference_import_web.py dient dazu, Webseiten zu crawlen und spezifische Metadaten oder Ressourcen (wie z.B. href, src, poster oder srcset-Attribute) aus den HTML-Quellen zu extrahieren. Es ist ein Werkzeug zur automatisierten Sammlung von Referenzen von Webseiten.

2. Abhängigkeiten

Das Skript verwendet standardmäßige Python-Bibliotheken.

  • argparse: Für die Verwaltung von Kommandozeilen-Argumenten.
  • html: Für die grundlegende HTML-Verarbeitung.
  • json: Für die mögliche Serialisierung oder Verarbeitung von Daten.
  • re: Für die regelbasierte Extraktion von Daten aus HTML-Strings (Regular Expressions).
  • shutil: Für Dateisystemoperationen (z.B. das Kopieren von Dateien).
  • subprocess: Für das Ausführen externer Befehle (z.B. bei komplexeren Aufgaben).
  • sys: Für den Zugriff auf die System- und Skript-Informationen.
  • time: Für Zeitmessungen oder Verzögerungen.
  • dataclasses: Zur Erstellung von unveränderlichen Datenstrukturen (MirrorSource).
  • pathlib.Path: Für die moderne und objektorientierte Handhabung von Dateipfaden.
  • urllib.parse.urlsplit: Für die Zerlegung von URLs.

3. Schlüsselkomponenten und Klassen

3.1. Regular Expressions (Regex)

Das Skript definiert zwei spezifische Regex-Muster zur Extraktion von Attributwerten:

  • **ATTR_RE**:
  • Zweck: Extrahiert Werte für Attribute wie href, src, poster oder action.
  • Muster: r"(?P<attr>\b(?:href|src|poster|action)=)(?P<quote>[\"'])(?P<value>.*?)(?P=quote)"
  • Erklärung: Sucht nach dem Schlüsselwort (href, src, etc.), gefolgt von einem Anführungszeichen (" oder '), dem Wert (.*?), und dem schließenden Anführungszeichen.
  • **SRCSET_RE**:
  • Zweck: Extrahiert Werte für das Attribut srcset.
  • Muster: r"(?P<attr>\bsrcset=)(?P<quote>[\"'])(?P<value>.*?)(?P=quote)"
  • Erklärung: Spezifisch für das srcset-Attribut.

3.2. Datenstruktur: MirrorSource

@dataclass(frozen=True, slots=True)class MirrorSource:
label: str
url: str
  • Typ: Eine dataclass, die zur strukturierten Speicherung von extrahierten Referenzen verwendet wird.
  • **frozen=True**: Die Instanz ist unveränderlich (immutable), was die Datenintegrität erhöht.
  • **slots=True**: Reduziert den Speicherverbrauch der Instanz, indem es statische Attribute verwendet.
  • Attribute:
  • label (str): Eine beschreibende Bezeichnung für die Quelle.
  • url (str): Die vollständige URL der Webseite.

4. Funktionsweise (Impliziert)

Obwohl der Hauptlogikteil des Skripts (die eigentliche Crawling- oder Parsing-Funktion) in den bereitgestellten Codeausschnitten nicht vollständig sichtbar ist, impliziert die Struktur folgende Schritte:

  1. Argumentenverarbeitung: Das Skript verwendet argparse, um von der Kommandozeile spezifische Eingaben (z.B. URLs, Pfade) zu erhalten.
  2. URL-Verarbeitung: Es wird wahrscheinlich die Eingabe URL zerlegt (urlsplit).
  3. HTML-Download/Crawling: Die Webseite wird abgerufen.
  4. HTML-Parsing: Der HTML-Inhalt wird analysiert.
  5. Extraktion: Die definierten Regex-Muster (ATTR_RE, SRCSET_RE) werden auf den HTML-String angewendet, um relevante Attribute und deren Werte zu finden.
  6. Datenmodellierung: Die extrahierten Informationen werden in Instanzen der MirrorSource-Klasse gespeichert.
  7. Speicherung: Die gesammelten Referenzen werden in einer definierten Struktur (wahrscheinlich JSON oder eine Datei) persistent gespeichert.

5. Sicherheit und Robustheit

  • Regex-Spezifikation: Die Verwendung von re.IGNORECASE und re.DOTALL in den Regex-Kompilierungen erhöht die Robustheit gegenüber unterschiedlichen Formatierungen (z.B. unterschiedliche Groß-/Kleinschreibung oder Zeilenumbrüche im Attributwert).
  • Immutability: Die Verwendung von frozen=True für MirrorSource stellt sicher, dass die gesammelten Daten nicht versehentlich während des Verarbeitungsprozesses modifiziert werden.
  • Dateisystem-Handling: Die Verwendung von pathlib.Path deutet auf eine moderne und plattformübergreifende Dateihandhabung hin.

6. Zusammenfassung der Struktur

Das Skript ist modular aufgebaut, wobei die Regex-Definitionen die Kernlogik der Datenextraktion definieren, während die MirrorSource-Klasse die Struktur der gesammelten Ergebnisse formalisiert. Es ist ein typisches Beispiel für ein Web-Scraping- oder Daten-Extraktionswerkzeug.

1. Überblick

Die Klasse ReferenceWebMirrorBuilder ist eine Klasse, die für die Erstellung eines Web-Mirrors (oder einer Referenzseite) konzipiert ist. Sie orchestriert den Prozess des Herunterladens von Webseiten mithilfe von wget (oder einem entsprechenden Befehl) und führt nach dem Download eine umfangreiche HTML-Manipulation durch, um die Inhalte für eine spezifische Referenzstruktur zu formatieren. Die Hauptaufgaben der Klasse sind:

  1. Mirroring: Ausgeben von Inhalten von definierten Quellen (MirrorSource) mithilfe von wget.
  2. HTML-Transformation: Überarbeitung der heruntergeladenen HTML-Dateien, insbesondere von Attributen und srcset-Attributen.
  3. Landing Page Erstellung: Generierung einer speziellen Startseite (index.html) mit einer Liste der gefundenen Quellen.
  4. Manifest-Erstellung: Erstellung einer JSON-Datei, die alle Metadaten und die Durchführung der Operation zusammenfasst.

Diese Funktion orchestriert den gesamten Importprozess. Zweck: Die Funktion initialisiert den Argumentparser, validiert die Eingaben und führt die Generierung der Manifeste für ausgewählte Mirror-Packs durch. Parameter:

  • argv: Iterable[str] | None = None: Eine Optionale Iterable von Strings, die die Kommandozeilenargumente enthält. Wenn None ist, werden die Standardargumente verwendet.

    • Rückgabe:

    • Logik und Ablauf:

  1. Argumentparser Initialisierung:
  • parser = build_argument_parser(): Initialisiert den Argumentparser, um Kommandozeilenargumente zu parsen.
  • args = parser.parse_args(list(argv) if argv is not None else None): Parsed Argumente werden basierend auf den bereitgestellten argv verarbeitet.
  1. Argument Validierung:
  • if not args.all and not args.pack:: Prüft, ob entweder die Option --pack <slug> oder --all angegeben wurde.
  • parser.error("Bitte entweder --pack <slug> oder --all angeben."): Wenn keine der beiden Optionen vorhanden ist, wird eine Fehlermeldung durch den Parser ausgegeben.
  1. Konfiguration der Ausgabe:
  • output_root = Path(args.output_root): Definiert den Wurzelpfad für die Ausgabe der generierten Manifeste.
  1. **Auswahl der Pakete (selected):**
  • selected = sorted(MIRROR_PACKS) if args.all else [str(args.pack)]: Bestimmt die Liste der zu bearbeitenden slugs (Schlüsselnamen), basierend auf der gewählten Option:
  • Wenn --all verwendet wird, werden alle verfügbaren MIRROR_PACKS sortiert.
  • Wenn --pack <slug> verwendet wird, wird nur der spezifische slug aus der Eingabe verwendet.
  1. Manifest-Generierung (Hauptschleife):
  • manifests: dict[str, object] = {}: Initialisiert ein Dictionary, um die Ergebnisse der Manifest-Generierung zu speichern.
  • Die Schleife iteriert über die selected slugs:
  • pack = MIRROR_PACKS[slug]: Holt das entsprechende Paketobjekt.
  • builder = ReferenceWebMirrorBuilder(pack=pack, output_root=output_root / slug, clean=bool(args.clean)): Initialisiert den ReferenceWebMirrorBuilder mit dem Paket, dem Zielpfad und der Option zur Bereinigung (clean).
  • manifests[slug] = builder.finalize_existing_site() if args.finalize_only else builder.build(): Führt die eigentliche Generierungsoperation durch:
  • Wenn args.finalize_only wahr ist, wird builder.finalize_existing_site() aufgerufen.
  • Andernfalls wird builder.build() aufgerufen.
  1. Ausgabe:
  • print(json.dumps(manifests, ensure_ascii=False, indent=2)): Gibt das gesamte generierte manifests-Dictionary in einem gut lesbaren, formatierten JSON-Format auf die Standardausgabe aus.
  1. Exit-Punkt:
  • return 0: Gibt den erfolgreichen Abschluss zurück.
