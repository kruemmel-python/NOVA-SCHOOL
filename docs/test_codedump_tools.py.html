Technische Dokumentation: test_codedump_tools.py

1. Überblick

Diese Datei, test_codedump_tools.py, dient als Testsuite für die Funktionen der nova_school_server.codedump_tools-Module. Sie verwendet das unittest-Framework, um die Funktionalität der Tools zur Erfassung, Komprimierung und Formatierung von Code-Dumps zu validieren. Die Hauptziele dieser Tests sind:

  1. Validierung der Konfiguration: Sicherstellen, dass die Konfigurationen korrekt verarbeitet werden.
  2. Test der Datenerfassung: Überprüfung, ob collect_directory_dump die Dateien korrekt sammelt.
  3. Test der Ausgabeformatierung: Überprüfung, ob dump_target_to_markdown die erfassten Daten korrekt in das gewünschte Markdown-Format umwandelt.

2. Abhängigkeiten

Die Datei ist abhängig von folgenden Modulen und Bibliotheken:

  • **tempfile**: Für das Erstellen temporärer Dateien und Verzeichnisse während der Tests.
  • **unittest**: Das Standard-Framework für die Durchführung der Tests.
  • **zipfile**: Für die Überprüfung der Komprimierungsfunktionen.
  • **pathlib.Path**: Für das moderne und objektorientierte Umgang mit Dateipfaden.
  • **nova_school_server.codedump_tools**: Importiert die Kernfunktionen der Tools, darunter:
  • DumpConfig
  • collect_directory_dump
  • config_for_profile
  • default_output_path_for_profile
  • dump_target_to_markdown

Technische Dokumentation: test_codedump_tools.py

1. Überblick

Diese Datei enthält eine Reihe von Unit-Tests (CodeDumpToolsTests), die die Funktionalität der Tools zur Erstellung von Code-Dumps (wahrscheinlich dump_target_to_markdown, collect_directory_dump, default_output_path_for_profile) validieren. Die Tests prüfen verschiedene Szenarien, wie z.B. die Ignorierung von Laufzeitdaten, die Verarbeitung von ZIP-Archiven, die Erkennung großer Dateien und die Anwendung verschiedener Konfigurationen (Profile) zur Selektion von Dateien.

2. Struktur und Komponenten

Die Datei implementiert eine Testklasse CodeDumpToolsTests, die spezifische Testfälle definiert, um die Robustheit der Code-Dumping-Funktionalität zu gewährleisten.

2.1. Testfälle (Test Methods)

Die folgenden Methoden testen spezifische Aspekte der Code-Dumping-Logik:

2.1.1. test_directory_dump_ignores_runtime_data_and_writes_markdown

  • Ziel: Überprüft, dass die Funktion bei der Erstellung eines Markdown-Dumps von einem Verzeichnis:
  • Laufzeitdaten ignoriert.
  • Dateien korrekt in Markdown-Formatierung dargestellt werden.
  • Unterordner und spezifische Dateien (wie index.json in einem Datenverzeichnis) korrekt ausgenommen werden.
  • Setup: Erstellt ein temporäres Verzeichnis mit Python-Dateien (main.py) und einer Markdown-Datei (README.md), sowie ein verschachteltes Datenverzeichnis (data/reference_library/index.json).
  • Erwartetes Verhalten: Der generierte CODEDUMP.md muss die Hauptdateien enthalten, aber die JSON-Datei im tiefsten Verzeichnis darf nicht erscheinen.

2.1.2. test_zip_dump_keeps_zip_support

  • Ziel: Validiert, dass die Funktion korrekt mit ZIP-Archiven umgehen kann und die Inhalte dieser Archive korrekt in den Markdown-Dump übertragen werden.
  • Setup: Erstellt eine ZIP-Datei mit zwei Dateien (src/app.py und data/reference_library/index.json).
  • Erwartetes Verhalten: Der Dump muss die Inhalte der ZIP-Datei korrekt abbilden, wobei die Struktur und die Ignorierung von bestimmten Pfaden beibehalten werden.

2.1.3. test_directory_collect_marks_large_files_with_placeholder

  • Ziel: Testet die Funktion collect_directory_dump zur Erkennung von Dateien, die über eine definierte Größe hinausgehen.
  • Setup: Erstellt eine große Python-Datei (big.py).
  • Erwartetes Verhalten: Die Funktion muss die Datei als zu groß markieren und einen Platzhaltertext ("file too large") im Ergebnis zurückgeben.

2.1.4. test_compact_profile_excludes_docs_wiki_and_tests

  • Ziel: Testet ein spezifisches Profil (compact), das Dokumentation (docs), Wiki-Inhalte und Tests (tests) aus dem Dump ausschließt.
  • Setup: Erstellt eine Struktur mit Code (server.py), Tests (tests/test_demo.py), Dokumentation (docs/guide.md) und Wiki (wiki/teacher.md).
  • Erwartetes Verhalten: Der Dump sollte nur den Code (server.py) enthalten und die Dokumentations- und Testdateien nicht.

2.1.5. test_deep_profile_includes_docs_wiki_and_tests

  • Ziel: Testet ein Profil (deep), das im Gegensatz zum compact-Profil Dokumentation, Wiki und Tests einschließt.
  • Setup: Gleiches Setup wie in test_compact_profile_excludes_docs_wiki_and_tests, aber mit der Konfiguration für das deep-Profil.
  • Erwartetes Verhalten: Der Dump muss alle Dateien (Code, Docs, Wiki, Tests) enthalten.

2.1.6. test_default_output_path_uses_profile_suffix_for_non_standard_profiles

  • Ziel: Überprüft die Logik zur Generierung des Ausgabe-Pfades basierend auf dem Profilnamen und dem Zielort (Verzeichnis oder ZIP-Datei).
  • Setup: Definiert spezifische Zielpfade (directory_target, zip_target).
  • Erwartetes Verhalten: Die erwarteten Namenskonventionen für die generierten Dump-Dateien (CODEDUMP.md, CODEDUMP.compact.md, project.deep.codedump.md) werden überprüft.

2.1.7. test_existing_dump_artifacts_are_not_reincluded

  • Ziel: Stellt sicher, dass beim Erstellen eines neuen Dumps bestehende, bereits generierte Dump-Artefakte (z.B. CODEDUMP.compact.md) nicht versehentlich wieder aufgenommen werden.
  • Setup: Erstellt einen alten Dump (CODEDUMP.compact.md) und versucht, einen neuen, tieferen Dump (CODEDUMP.deep.md) zu erstellen.
  • Erwartetes Verhalten: Der neue Dump sollte die neuen Informationen enthalten, aber die alten Artefakte dürfen nicht wieder auftauchen.

3. Abhängigkeiten und Voraussetzungen

  • Standardbibliothek: unittest, tempfile, pathlib, zipfile.
  • Externe Funktionen/Klassen:
  • dump_target_to_markdown(source, output_path, config): Die zentrale Funktion zur Erstellung des Markdown-Dumps.
  • collect_directory_dump(root, config): Funktion zur Sammlung von Dateien aus einem Verzeichnis.
  • default_output_path_for_profile(target, profile): Funktion zur Bestimmung des korrekten Ausgabeformats.
  • config_for_profile(profile_name): Funktion zur Erstellung von Konfigurationsobjekten für verschiedene Profile.

4. Zusammenfassung der Testziele

Die Testsuite stellt sicher, dass die Code-Dumping-Bibliothek folgende Kernanforderungen erfüllt:

  1. Korrekte Datenextraktion: Dateien werden korrekt identifiziert und extrahiert (inkl. ZIP-Support).
  2. Konfigurationsgesteuerte Selektion: Profile ermöglichen präzise Steuerung, welche Teile des Projekts (Code, Docs, Tests) im Dump enthalten sind.
  3. Robustheit: Umgang mit Randfällen wie sehr großen Dateien oder bestehenden Artefakten.
  4. Formatierung: Der finale Output ist konsistent im Markdown-Format.
