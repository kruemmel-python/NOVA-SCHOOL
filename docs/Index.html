Projekt-Architektur

Analyse

Basierend auf den Dokumentationen der einzelnen Komponenten lässt sich das Projekt als ein komplexes, mehrschichtiges System definieren, das sich auf die Automatisierung von Analyseprozessen, die Verwaltung von Code-Ausführungen, die Erstellung von Dokumentationen/Lehrplänen und die Orchestrierung von Deployment-Pipelines konzentriert. ---

Projektübersicht: Intelligente Analyse- und Deployment-Plattform

1. Hauptzweck des Projekts

Das Hauptziel dieses Projekts ist die Schaffung einer zentralisierten, automatisierte Plattform, die den gesamten Lebenszyklus von Software-Analyse, Code-Ausführung, Dokumentationsgenerierung und Deployment-Orchestrierung abdeckt. Das System soll folgende Kernfunktionen erfüllen:

  • Intelligente Code-Interaktion: Bereitstellung eines zentralen Dienstes (ai_service.py) zur Interaktion mit einem lokalen Code-Assistenten, der auf Basis von Large Language Models (LLMs) arbeitet.
  • Analyse- und Archivierung: Automatisierung des Prozesses zur Identifizierung, Filterung und Archivierung von Analyseergebnissen (analysis_archive_builder.py).
  • Code-Orchestrierung: Zentrales Management des Ausführens von Code in verschiedenen Umgebungen (Container) basierend auf der Programmiersprache (code_runner.py).
  • Dokumentations- und Curriculum-Management: Strukturierte Verwaltung von Lehrplänen, Fragenstrukturen und der Erstellung von Zertifikaten/Dokumentationen (curriculum.*, docs_catalog.py).
  • Infrastruktur- und Konfigurationsmanagement: Bereitstellung robuster Mechanismen für Authentifizierung (auth.py), Konfiguration (config.py), Container-Sicherheit (container_seccomp.py) und die Verwaltung von verteilten Prozessen (distributed.py).
  • Deployment-Pipeline: Orchestrierung komplexer Deployment-Schritte (deployments.py, distribution_builder.py).

    • Zusammenfassend: Das Projekt zielt darauf ab, die manuelle, repetitive Arbeit in der Softwareentwicklung und im Projektmanagement durch KI-gestützte Automatisierung und robuste Orchestrierung zu ersetzen.

2. Technologische Stack-Wahl

Die technologische Architektur ist stark auf Python als primäre Sprache und Orchestrierungs-Runtime ausgerichtet, was durch die Vielzahl an spezialisierten Modulen bestätigt wird.

Kategorie Schlüsseltechnologien / Komponenten Begründung der Wahl
Sprache & Runtime Python Hohe Eignung für Datenverarbeitung, Machine Learning (LLM-Integration), Skripting und die schnelle Entwicklung von Orchestrierungslogik.
KI / NLP Large Language Models (LLMs) Zentraler Bestandteil für die Interaktion mit dem Code-Assistenten (ai_service.py).
Containerisierung Docker / Container-Orchestrierung Notwendig für die Ausführung von Code in isolierten Umgebungen (code_runner.py) und für das Deployment.
Datenbank / Speicherung Implizit (durch database.py) Erforderlich für die Speicherung von Konfigurationen, Benutzerdaten (auth.py) und Analyseergebnissen.
Konfiguration JSON / YAML (über config.py) Standardisierte Methode zur Verwaltung von Umgebungsvariablen und Anwendungseinstellungen.
Archivierung / Dateisystem ZIP-Archivierung (archive_utils.py) Für die sichere und portable Speicherung von Analyseergebnissen.
Orchestrierung Modulare Struktur (z.B. deployments.py, distributed.py) Die Architektur ist stark auf die Orchestrierung von sequenziellen und parallelen Aufgaben ausgelegt.

3. Datenfluss (Data Flow)

Der Datenfluss im System ist modular und iterativ, wobei Daten zwischen den spezialisierten Modulen fließen, um eine end-to-end-Automatisierung zu ermöglichen.

A. Analyse- und KI-Datenfluss (Input $\rightarrow$ Processing)

  1. Input: Rohdaten oder Code-Snippets werden in das System eingespeist.
  2. LLM-Interaktion: Der ai_service.py empfängt die Anfrage und interagiert mit dem LLM, um eine intelligente Analyse oder Code-Vorschläge zu generieren.
  3. Analyse-Pipeline: Die Ergebnisse dieser Analyse werden an analysis_archive_builder.py weitergeleitet, welches die Ergebnisse filtert, strukturiert und in ein Archivformat vorbereitet.

B. Code-Ausführungs- und Deployment-Datenfluss (Processing $\rightarrow$ Execution)

  1. Code-Erkennung: code_runner.py analysiert eine Datei, um die Programmiersprache zu identifizieren.
  2. Umgebungs-Setup: Basierend auf der Sprache und den Konfigurationen aus config.py wird die notwendige Umgebung (Container-Definition) vorbereitet.
  3. Ausführung: Der Code wird im Container ausgeführt.
  4. Distribution: Die Ergebnisse oder Artefakte des Code-Runs werden durch distribution_builder.py verarbeitet, um die finale Distribution zu erstellen.
  5. Deployment: deployments.py orchestriert die Schritte, um die fertige Distribution in die Zielumgebung zu bringen.

C. Metadaten- und Benutzerdatenfluss (Management)

  1. Authentifizierung: Benutzeranfragen werden über auth.py validiert, um den aktuellen Session Context zu erstellen und Benutzerinformationen zu extrahieren.
  2. Konfigurations-Abfrage: Alle Module greifen auf config.py zu, um notwendige Parameter für ihre spezifischen Aufgaben (z.B. Pfade, API-Keys) abzurufen.
  3. Dokumenten-Generierung: Daten aus dem Curriculum-Modul (curriculum.*) werden verarbeitet (Textvorverarbeitung in curriculum_certificate_pdf.py) und in die zentrale Dokumentationsverwaltung (docs_catalog.py) eingespeist.
