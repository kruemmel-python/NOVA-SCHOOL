Technische Dokumentation: test_permissions.py

1. Überblick

Diese Datei enthält eine Reihe von Unit-Tests, die die Logik zur Auflösung von Benutzerrechten (resolve_permissions) für verschiedene Rollen (z.B. "student", "teacher", "admin") und die Anwendung von Übertreibungen (Overrides) durch Benutzer und Gruppen validieren. Die Tests stellen sicher, dass die Berechtigungen gemäß den definierten Regeln korrekt zugewiesen werden, insbesondere im Hinblick auf Standardeinstellungen, Rollen-Spezifika und die Priorisierung von Benutzer-Overrides.

2. Struktur und Komponenten

Die Datei definiert eine Testklasse PermissionTests, die spezifische Szenarien für die Berechtigungszuweisung abdeckt.

2.1. Hauptfunktion: resolve_permissions(role, group_overrides=None, user_overrides=None)

Obwohl die Implementierung dieser Funktion nicht in der bereitgestellten Datei sichtbar ist, wird sie durch die Tests impliziert. Sie ist die zentrale Funktion, die die endgültige Berechtigungsstruktur basierend auf der Rolle, optionalen Gruppen-Overrides und Benutzer-Overrides generiert.

2.2. Testfälle

Die folgenden Tests validieren spezifische Verhaltensweisen:

2.2.1. test_student_defaults_enable_ai_but_disable_web

Ziel: Überprüfen, dass die Standardeinstellungen für die Rolle "student" korrekt angewendet werden. Erwartete Ergebnisse:

  • ai.use: True (AI-Funktionen sind aktiviert).
  • mentor.use: True.
  • teacher.materials.use: False (Materialien sind für Studenten deaktiviert).
  • curriculum.use: True.
  • curriculum.update: False (Aktualisierungen sind deaktiviert).
  • web.access: False (Webzugriff ist deaktiviert).
  • run.python: True (Python-Ausführung ist erlaubt).

2.2.2. test_teacher_defaults_do_not_gain_curriculum_update

Ziel: Überprüfen, dass die Standardeinstellungen für die Rolle "teacher" korrekt sind, insbesondere im Hinblick auf die Aktualisierung des Curriculums. Erwartete Ergebnisse:

  • curriculum.manage: True.
  • curriculum.update: False (Wichtig: Teacher erhält keine automatische Update-Berechtigung).
  • admin.manage: False.
  • teacher.materials.use: True.
  • teacher.chat.moderate: True.

2.2.3. test_user_override_wins_over_group_false

Ziel: Validieren, dass ein spezifischer Benutzer-Override die Berechtigungen über Gruppen-Overrides hinweg überschreibt. Szenario:

  • Rolle: "student"
  • Group Overrides: {"run.python": False, "chat.use": True}
  • User Overrides: {"run.python": True}

    • Erwartete Ergebnisse:

  • run.python: True (User Override True über Group Override False gewinnt).
  • chat.use: True (User Override True über Group Override True gewinnt, hier ist die Konsistenz gegeben).

2.2.4. test_admin_role_keeps_core_permissions_even_if_user_override_disables_them

Ziel: Sicherstellen, dass Kernberechtigungen einer hochrangigen Rolle ("admin") erhalten bleiben, selbst wenn der Benutzer versucht, diese spezifischen Berechtigungen über user_overrides zu deaktivieren. Szenario:

  • Rolle: "admin"
  • User Overrides: {"admin.manage": False, "chat.use": False, "docs.read": False}

    • Erwartete Ergebnisse:

  • admin.manage: True (Admin-Kernberechtigung wird beibehalten).
  • chat.use: True (Admin-Kernberechtigung wird beibehalten).
  • docs.read: True (Admin-Kernberechtigung wird beibehalten).
  • teacher.materials.use: True (Dies deutet darauf hin, dass Admin-Rollen bestimmte, nicht-überflutete Berechtigungen erhalten).

3. Zusammenfassung der Testziele

Test-ID Fokus Validierter Mechanismus Schlüsselprinzip
test_student_defaults_enable_ai_but_disable_web Standard-Rollen-Konfiguration Standard-Rollen-Definiert Baseline-Sicherheit

test_user_override_wins_over_group_false Priorisierung von Overrides Hierarchie: User > Group Prioritätsregeln

4. Fazit

Die Testsuite in test_permissions.py ist effektiv darin, die komplexe Logik der Berechtigungsverwaltung zu validieren. Sie deckt die wichtigsten Interaktionen zwischen Rollen-Defaults, Gruppen-Konfigurationen und Benutzer-spezifischen Übertreibungen ab. Die Tests bestätigen, dass die Priorisierungsregeln (User > Group) und die Schutzmechanismen für hochrangige Rollen (Admin) korrekt implementiert sind.
