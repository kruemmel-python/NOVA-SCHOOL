Technische Dokumentation: embedded_nova.py

1. Überblick

Diese Datei (embedded_nova.py) definiert die Struktur für ein EmbeddedPrincipal-Objekt, das wahrscheinlich zur Verwaltung von Benutzer- oder Tenant-Identitäten innerhalb eines Systems verwendet wird. Die Datei enthält auch eine Hilfsfunktion zur sicheren Serialisierung von Daten in JSON-Format und die Definition einer Datenstruktur.

2. Funktionen

_json_safe(value: Any) -> Any

Zweck: Diese interne Hilfsfunktion dient dazu, komplexe Python-Datenstrukturen (wie Dictionaries, Listen, Tuples und Sets) sicher in ein Format zu konvertieren, das für die JSON-Serialisierung geeignet ist. Sie stellt sicher, dass spezielle Python-Typen wie pathlib.Path korrekt als Strings behandelt werden. Parameter:

  • value: Any: Das zu serialisierende Python-Objekt (Dictionary, Liste, Tupel, Set oder Path).

    • Rückgabe:

  • Any: Das transformierte oder unveränderte Python-Objekt.

    • Details:

  • Dictionaries: Konvertiert Schlüssel zu Strings (str(key)), um sicherzustellen, dass JSON-Schlüssel strings sind.
  • Listen/Tuples: Rekursiv durchläuft die Elemente und wendet _json_safe darauf an.
  • Sets: Konvertiert das Set in eine sortierte Liste, da Sets in JSON nicht nativ unterstützt werden.
  • **pathlib.Path:** Konvertiert Path-Objekte explizit in Strings.
  • Andere Typen: Gibt den Wert unverändert zurück.

3. Klassen

EmbeddedPrincipal

Zweck: Diese dataclass definiert die Struktur für eine primäre Identität oder ein Hauptprofil innerhalb des eingebetteten Systems. Sie hält alle notwendigen Informationen über den Benutzer/Tenant und seine Berechtigungen. Attribute:

Attribut Typ Beschreibung
tenant_id str Die eindeutige Kennung des Tenants oder der Organisation, zu der der Principal gehört.
token_id str Eine eindeutige Identifikations- oder Sitzungstoken für diesen Principal.
subject str Der Subjektname oder die eindeutige Identifikation des Principals.
roles list[str] Eine Liste von Strings, die die zugewiesenen Rollen oder Berechtigungen des Principals darstellen.
metadata dict[str, Any] Ein flexibles Dictionary zur Speicherung von zusätzlichen, nicht strukturell definierten Metadaten.
expires_at float Der Zeitpunkt (wahrscheinlich als Unix-Timestamp) für das Ablaufdatum dieses Principals.

Spezifikationen:

  • **slots=True:** Wird verwendet, um die Speicherplatznutzung zu optimieren, indem die Verwendung von __dict__ vermieden wird.
  • Typisierung: Nutzt moderne Python-Typen (typing Module) für bessere Code-Readability und statische Analyse.

4. Abhängigkeiten

  • json: Für die (implizierte) Serialisierung/Deserialisierung von Daten.
  • secrets: Wahrscheinlich für die Generierung von sicheren Tokens oder Schlüssel.
  • threading: Zeigt an, dass die Klasse oder die Umgebung Multithreading unterstützt.
  • time: Wird wahrscheinlich für Zeitmessungen oder die Verarbeitung von Zeitstempeln verwendet.
  • uuid: Wird wahrscheinlich für die Generierung eindeutiger Identifikatoren verwendet.
  • dataclasses: Für die einfache Definition von Datenstrukturen.
  • pathlib.Path: Für die Verwaltung von Dateipfaden.
  • typing: Für fortgeschrittene Typisierung.

5. Nutzungsszenarien (Impliziert)

Die Klasse EmbeddedPrincipal wird typischerweise verwendet, um:

  1. Authentifizierung: Die Identität eines Benutzers oder Dienstes zu repräsentieren.
  2. Autorisierung: Die zugewiesenen roles zu überprüfen, um den Zugriff auf Ressourcen zu kontrollieren.
  3. Datenübertragung: Die Daten in ein JSON-Format zu konvertieren, wobei die _json_safe Funktion sicherstellt, dass alle komplexen Typen korrekt serialisiert werden.

Technische Dokumentation: EmbeddedSecurityPlane Klasse

1. Überblick

Die Klasse EmbeddedSecurityPlane dient als zentraler Mechanismus zur Verwaltung aller Sicherheitsressourcen, Benutzer (Tenants), Tokens, geheimen (Secrets), Zertifizierungsstellen (CAs) und Worker-Enrollments innerhalb einer eingebetteten Sicherheitsarchitektur. Sie ist zustandsorientiert und speichert ihren gesamten Zustand persistent in einer JSON-Datei.

2. Klasse: EmbeddedSecurityPlane

2.1. Konstruktor (__init__)

Der Konstruktor initialisiert die Klasse, indem er den Pfad zur Basisdatenbank festlegt und den aktuellen Zustand lädt. Parameter:

  • base_path: Path: Der Pfad zur Basisverzeichnis, in dem die Daten gespeichert werden sollen.

    • Funktionalität:

  1. Pfadauflösung: Der base_path wird aufgelöst (resolve(strict=False)).
  2. State-Pfad: Der Pfad zur Speicherdatei (state_path) wird definiert: base_path/data/embedded_security_plane.json.
  3. Verzeichnisstruktur: Das Verzeichnis data/ wird sicher erstellt (mkdir(parents=True, exist_ok=True)).
  4. Locking: Ein threading.RLock (_lock) wird initialisiert, um den Zugriff auf den internen Zustand zu synchronisieren.
  5. Zustandsinitialisierung: Der aktuelle Zustand wird über _load_state() geladen.

2.2. Private Methoden (Intern)

_load_state() -> dict[str, Any]

Ladet den aktuellen Zustand aus der state_path.

  • Logik: Prüft, ob die Datei existiert. Wenn ja, versucht, den Inhalt als JSON zu parsen. Bei Fehlern (z.B. fehlerhaftes JSON) wird der leere Zustand (_empty_state()) zurückgegeben.

_save_state() -> None

Speichert den aktuellen internen Zustand (self._state) persistent.

  • Logik: Nutzt eine temporäre Datei (.tmp) zur Speicherung des JSON-Inhalts und führt anschließend einen replace()-Vorgang durch, um die Hauptdatei (state_path) zu aktualisieren. Dies minimiert das Risiko von Datenkorruption bei Schreibvorgängen.

_empty_state() -> dict[str, dict[str, Any]]

Stellt den Standardzustand zurück, falls die Datei nicht existiert oder fehlerhaft ist.

  • Inhalt: Ein Dictionary mit allen erwarteten Schlüsselspalten (tenants, tokens, secrets, certificate_authorities, trust_policies, worker_enrollments) mit leeren Unterstrukturen.

_secret_key(tenant_id: str, name: str) -> str

Generiert einen eindeutigen Schlüssel für die Speicherung von Secrets.

  • Format: f"{tenant_id}::{name}"

# Technische Dokumentation: `EmbeddedToolSandbox` und `EmbeddedNovaAIProviderRuntime`
## 1. Einführung
Diese Dokumentation beschreibt die Klasse `EmbeddedToolSandbox` und die Dataclass `EmbeddedNovaAIProviderRuntime`, die in der bereitgestellten Python-Datei (`embedded_nova.py`) definiert sind. Diese Komponenten dienen wahrscheinlich der Verwaltung und Authentifizierung von Werkzeugen (Tools) innerhalb eines eingebetteten Systems oder einer spezifischen AI-Provider-Runtime.
## 2. Klasse: `EmbeddedToolSandbox`
Die Klasse `EmbeddedToolSandbox` ist verantwortlich für die Verwaltung von Berechtigungen (Autorisierung) für Benutzer oder Prinzipien (Principals) bezüglich der zugelassenen und angeforderten Werkzeuge. Sie verwendet einen `RLock` zur Sicherstellung der Thread-Sicherheit bei der Verwaltung der Protokolle.
### 2.1. Konstruktor (`__init__`)

def __init__(self) -> None: self._lock = threading.RLock() self._recent: list[dict[str, Any]] = []

* **`self._lock`**: Ein `threading.RLock` (Reentrant Lock), der verwendet wird, um den Zugriff auf interne Zustände (insbesondere die Liste `self._recent`) synchron zu halten und Race Conditions zu verhindern.
* **`self._recent`**: Eine Liste, die alle kürzlich durchgeführten Autorisierungsanfragen (Payloads) speichert. Sie dient zur Protokollierung und zur Begrenzung der Größe der Historie.
### 2.2. Methode: `authorize`
Diese Methode führt den Prozess der Autorisierung durch und speichert die Anfrage in der Historie.
**Signatur:**

def authorize( self, principal: str, *, allowed_tools: set[str] | list[str] | tuple[str, ...] | None = None, requested_tools: set[str] | list[str] | tuple[str, ...] | None = None, metadata: dict[str, Any] | None = None, ) -> dict[str, Any]:

**Beschreibung:**
Diese Methode erstellt ein JSON-ähnliches Payload, das die Autorisierungsdetails enthält, und speichert es in der internen Historie.
**Parameter:**
* **`principal` (str)**: Der Identifikator des Principals (z.B. Benutzername, Service-ID).
* **`allowed_tools` (optional: `set[str] | list[str] | tuple[str, ...] | None`)**: Eine Sammlung von Werkzeugen, die dem `principal` explizit zugelassen sind.
* **`requested_tools` (optional: `set[str] | list[str] | tuple[str, ...] | None`)**: Eine Sammlung von Werkzeugen, die der `principal` aktuell anfordert.
* **`metadata` (optional: `dict[str, Any] | None`)**: Zusätzliche Metadaten, die mit der Autorisierungsanfrage verknüpft werden sollen.
**Rückgabe:**
* **`dict[str, Any]`**: Das generierte Autorisierungs-Payload.
**Interner Ablauf:**
1. Das Payload wird erstellt, wobei `allowed_tools` und `requested_tools` in Sets umgewandelt und sortiert werden, um Konsistenz zu gewährleisten.
2. Der Zugriff auf `self._recent` wird durch `self._lock` gesichert.
3. Das generierte Payload wird zur Liste `self._recent` hinzugefügt.
4. **Historienmanagement:** Wenn die Anzahl der Einträge in `self._recent` mehr als 50 beträgt, werden die ältesten 50 Einträge entfernt, um die Liste auf die letzten 50 Einträge zu beschränken.
### 2.3. Methode: `snapshot`
Diese Methode liefert einen aktuellen Überblick über den Zustand der Sandbox.
**Signatur:**

def snapshot(self) -> dict[str, Any]:

**Beschreibung:**
Gibt eine Zusammenfassung der aktuellen Sandbox-Zustände zurück.
**Rückgabe:**
* **`dict[str, Any]`**: Ein Dictionary, das folgende Informationen enthält:
* `"backend"`: Der Name des Backends (fest auf `"embedded"` gesetzt).
* `"recent_authorizations"`: Die aktuelle Anzahl der gespeicherten Autorisierungsanfragen in `self._recent`.

## 3. Dataclass: `EmbeddedNovaAIProviderRuntime`
Diese Dataclass dient zur Definition des Zustands oder der Konfiguration der Laufzeitumgebung für den Embedded Nova AI Provider.
### 3.1. Definition

@dataclass(slots=True) class EmbeddedNovaAIProviderRuntime: provider_id: str = "embedded"

**Eigenschaften:**
* **`provider_id` (str)**: Der eindeutige Identifikator des Providers.
* **Standardwert:** `"embedded"`.
### 3.2. Methode: `snapshot`

def snapshot(self) -> dict[str, Any]: return asdict(self)

**Beschreibung:**
Diese Methode serialisiert den aktuellen Zustand der `EmbeddedNovaAIProviderRuntime` in ein Dictionary.
**Rückgabe:**
* **`dict[str, Any]`**: Ein Dictionary, das alle Attribute der Instanz enthält.

## 4. Zusammenfassung der Struktur
| Komponente | Typ | Hauptzweck | Schlüsselfunktionen | Synchronisation |
| :--- | :--- | :--- | :--- | :--- |
| `EmbeddedToolSandbox` | Klasse | Verwaltung von Tool-Autorisierungen und Protokollierung. | `authorize()`, `snapshot()` | `threading.RLock` |
| `EmbeddedNovaAIProviderRuntime` | Dataclass | Definition des Zustands des AI-Provider-Runtimes. | `snapshot()` | Keine (statisch) |
## 5. Anmerkungen und Hinweise
1. **Typisierung:** Die Verwendung von `set` und `list` in den Signaturdefinitionen deutet auf eine flexible Eingabe von Tool-Satzungen hin.
2. **Datenintegrität:** Die `authorize`-Methode stellt sicher, dass die gespeicherten Daten (Payloads) konsistent formatiert werden (z.B. durch Sortierung der Tools).
3. **Performance:** Die Verwendung von `slots=True` in der Dataclass `EmbeddedNovaAIProviderRuntime` kann zu einer effizienteren Speicherung von Instanzdaten führen.
4. **Abhängigkeiten:** Die Dokumentation impliziert die Abhängigkeit von `threading` für die Synchronisation und von `dataclasses` sowie `json` (impliziert durch `_json_safe` in `authorize`) für die Datenmanipulation.
