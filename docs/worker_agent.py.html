Technische Dokumentation: worker_agent.py

1. Überblick

Die Datei worker_agent.py definiert die Hauptlogik für einen Agenten, der wahrscheinlich Aufgaben verteilt, Container verwaltet und möglicherweise mit externen Diensten kommuniziert. Der Code beinhaltet Funktionen für Argumentenpar parsing, Hash- und HMAC-Verifizierung, Dateioperationen (wie das Entpacken von ZIP-Dateien) und die Verwaltung von Worker-Dispatches.

2. Abhängigkeiten

Die Datei importiert Funktionen und Klassen aus anderen Modulen:

  • archive_utils.extract_zip_safely: Für das sichere Entpacken von ZIP-Dateien.
  • container_seccomp.resolve_seccomp_profile_option: Für die Auflösung von Seccomp-Profiloptionen.
  • worker_dispatch.RemoteWorkerDispatchService: Für die Verwaltung der Remote-Verteilung von Aufgaben an Worker-Prozesse.

3. Konstanten und Globale Definitionen

_BLOCKED_CONTAINER_ENV_KEYS

Dies ist ein Set von Umgebungsvariablen-Schlüsseln, die in Containern blockiert werden sollen. Diese Liste enthält Pfade und Konfigurationsvariablen, die potenziell Sicherheitsrisiken darstellen oder die Umgebung des Containers unnötig erweitern könnten. Zweck: Sicherstellung einer reduzierten und sicheren Umgebung für den Agenten/Container. Beispiel-Schlüssel:

  • path
  • prompt
  • homedrive
  • onedrive

4. Funktionen

_container_file_size_limit_bytes(raw_value: Any) -> str

Beschreibung: Diese interne Hilfsfunktion konvertiert einen gegebenen Wert (der möglicherweise eine Zahl oder eine Zeichenkette ist) in eine Zeichenkette, die die maximale Dateigröße in Bytes darstellt. Logik:

  1. Versucht, den Input als Integer zu konvertieren. Wenn dies fehlschlägt, wird ein Standardwert (65536 Bytes) verwendet.
  2. Wenn eine Zahl erfolgreich konvertiert wird, wird sie mit 1 multipliziert, um sie in Kilobytes (KB) zu erhalten, und dann mit 1024 multipliziert, um sie in Bytes umzurechnen.
  3. Stellt sicher, dass der Wert mindestens 1 Byte beträgt (max(1, limit_kb)).
  4. Gibt das Ergebnis als Zeichenkette zurück.

    5. Rückgabetyp: str

5. Klasse

(Die Klasse selbst ist in der bereitgestellten Snippet-Datei nicht vollständig definiert, aber sie ist der Hauptfokus der Datei.) Hinweis: Die Klasse, die mit <|end|> beginnt, ist der Kern des Agenten-Logik. Sie wird wahrscheinlich die Hauptmethoden für die Initialisierung, die Verarbeitung von Eingaben und die Interaktion mit den Worker-Dispatches enthalten.

6. Zusammenfassung der Hauptfunktionen

Die Datei dient als Schnittstelle für die Agenten-Logik, wobei sie sich auf:

  1. Sicherheit/Konfiguration: Durch die Definition von blockierten Umgebungsvariablen.
  2. Dateiverarbeitung: Durch die Nutzung von archive_utils zur sicheren Entpackung.
  3. Parallelität/Verteilung: Durch die Integration des RemoteWorkerDispatchService zur Aufgabenverteilung.
  4. Datenintegrität: Durch die Verwendung von hashlib und hmac (impliziert durch die Importe) zur Verifizierung von Daten oder Signaturprüfungen.

Technische Dokumentation: worker_agent.py

1. Überblick

Die Datei worker_agent.py definiert den Haupt-Entrypoint und die Konfiguration für einen Remote Worker Agent im Kontext von "Nova School". Dieser Agent ist dafür verantwortlich, sich mit einem zentralen Server zu verbinden, seine Identität zu authentifizieren und eine kontinuierliche Verbindung (Heartbeat) aufrechtzuerhalten. Der Agent wird über die Kommandozeile gestartet und benötigt spezifische Parameter, um seine Konfiguration zu definieren.

2. Funktionen

2.1. _default_advertise_host() -> str

Diese interne Funktion dient dazu, einen geeigneten Hostnamen oder IP-Adresse zu bestimmen, der für andere Worker zur Erreichbarkeit dieses Nodes dient. Beschreibung: Versucht, eine externe IP-Adresse zu ermitteln, indem es versucht, eine Verbindung zum Google DNS Server (8.8.8.8:80) herzustellen. Wenn eine externe IP erfolgreich abgerufen werden kann und nicht lokal (127.x.x.x) ist, wird diese IP als advertise_host zurückgegeben. Falls der Versuch fehlschlägt, wird der lokale Hostname des Systems zurückgegeben. Rückgabetyp: str (String) Wirkungsweise:

  1. Versucht, eine TCP-Verbindung zum Ziel ("8.8.8.8", 80) herzustellen.
  2. Wenn die Verbindung erfolgreich ist, wird die zurückgegebene Adresse (probe.getsockname()[0]) geprüft.
  3. Wenn die Adresse nicht lokal beginnt, wird sie als Ergebnis zurückgegeben.
  4. Im Falle eines OSError (z.B. Timeout oder Verbindungsproblem) wird die Funktion einfach pass und der lokale Hostname zurückgegeben.

2.2. main() -> None

Dies ist die Hauptfunktion, die die Argumentenparalysierung und die Initialisierung des WorkerAgent-Objekts übernimmt. Beschreibung: Diese Funktion parst die Kommandozeilenargumente (argparse) und initialisiert den WorkerAgent mit den bereitgestellten Konfigurationsparametern. Sie startet anschließend die Hauptlaufschleife des Agents (agent.run_forever()). Argumente (Command-Line Arguments):

Argument Typ Erforderlich Beschreibung Standardwert
--server str Ja Die Basis-URL des Nova School Servers (z.B. http://192.168.178.62:8877). Keine
--worker-id str Ja Eindeutige Identifikationsnummer des Workers. Keine
--token str Ja Der Bootstrap-Token, der vom Server bereitgestellt wird. Keine
--advertise-host str Nein Hostname/IP, unter der andere Worker diesen Node erreichen sollen. Ergebnis von _default_advertise_host()
--work-root str Nein Lokales Arbeitsverzeichnis des Workers. $HOME/.nova-school-worker
--heartbeat-seconds float Nein Intervall in Sekunden für das Heartbeat-Signal. 3.0

Ablauf:

  1. Argumentenparalyse: Die Argumente werden gemäß den definierten Spezifikationen gelesen.
  2. Agent-Initialisierung: Ein Instanz des WorkerAgent wird erstellt, wobei alle geparsten Argumente in die entsprechenden Parameter des Konstruktors übergeben werden.
  3. Start: Die Methode agent.run_forever() wird aufgerufen, was den Hauptbetrieb des Agents startet.

2.3. if __name__ == "__main__": Block

Beschreibung: Dieser Standard-Block stellt sicher, dass die main() Funktion nur ausgeführt wird, wenn das Skript direkt als Programm gestartet wird (und nicht als Importmodul). Ablauf: Rufen die main() Funktion auf, um den Agenten zu starten.

3. Abhängigkeiten

Basierend auf dem Code werden folgende Standard-Python-Module benötigt:

  • socket: Für Netzwerkoperationen (Verbindung zum Server, Host-Erkennung).
  • argparse: Für die Verwaltung der Kommandozeilenargumente.
  • pathlib.Path: Für die Verwaltung von Dateipfaden (insbesondere für --work-root).
  • WorkerAgent (extern definiert): Eine Klasse, die die eigentliche Logik des Worker-Agents implementiert und die run_forever() Methode besitzt.

4. Zusammenfassung der Konfiguration

Der Agent wird durch die folgenden Schlüsselparameter gesteuert:

  • Server-Endpunkt: Wo der Agent seine Hauptkommunikation findet (--server).
  • Identität: Eindeutige Kennung (--worker-id).
  • Authentifizierung: Token zur Initialisierung (--token).
  • Netzwerk-Erreichbarkeit: Adresse, die für andere Worker relevant ist (--advertise-host).
  • Lokale Umgebung: Wo die Arbeitsdateien gespeichert werden (--work-root).
  • Kontrollfrequenz: Wie oft der Agent seine Vitalität meldet (--heartbeat-seconds).
