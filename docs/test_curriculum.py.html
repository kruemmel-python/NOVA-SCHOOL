Technische Dokumentation: test_curriculum.py

1. Überblick

Diese Datei, test_curriculum.py, scheint eine Sammlung von Testfunktionen und Klassen zu enthalten, die wahrscheinlich die Funktionalität eines Systems zur Verwaltung und Prüfung von Lehrplänen (Curricula) testen. Sie importiert Komponenten aus verschiedenen Modulen des nova_school_server-Frameworks, einschließlich Dienste, Katalogen und Datenbank-Repositories.

2. Importierte Module und Abhängigkeiten

Die Datei importiert die folgenden Module und Klassen:

  • **__future__**: Für zukünftige Python-Syntax-Funktionen.
  • **hashlib, hmac, io, json, tempfile, time, unittest, zipfile, pathlib.Path**: Standardmodulmodule für kryptografische Hashing, I/O-Operationen, temporäre Dateien, Zeitmessung, Unit-Testing und Dateiverwaltung.
  • **PIL.Image**: Für die Verarbeitung von Bildern (wahrscheinlich zur Darstellung von Kursmaterialien oder Aufgaben).
  • **nova_school_server.curriculum.CurriculumService**: Wahrscheinlich die Hauptklasse für die Geschäftslogik bezüglich des Lehrplans.
  • **nova_school_server.curriculum_catalog.get_course, list_courses**: Funktionen zur Abfrage und Auflistung von Kursinformationen.
  • **nova_school_server.database.SchoolRepository**: Eine Klasse zur Abstraktion der Datenbankoperationen.

3. Klassen und Funktionen

3.1. Klasse class _Session

Diese Klasse dient zur Speicherung von Kontextinformationen für eine Benutzer- oder Testsession. Attribute:

  • username: str: Der Benutzername der Session.
  • is_teacher: bool = False: Flag, ob der Benutzer ein Lehrer ist (Standardwert: False).
  • permissions: dict | None = None: Ein Dictionary mit den Berechtigungen des Benutzers (Standardwert: leeres Dictionary {}).
  • group_ids: list[str] | None = None: Eine Liste von Gruppen-IDs, zu denen der Benutzer gehört (Standardwert: leere Liste []).

    • **Konstruktor (__init__)**: Initialisiert die Session mit den bereitgestellten Benutzerdaten und Werten für die optionalen Attribute.

3.2. Funktion _correct_answers(questions: list[dict]) -> dict[str, object]

Diese interne Hilfsfunktion ist dafür verantwortlich, die korrekten Antworten basierend auf der Struktur von Fragen zu extrahieren. Parameter:

  • questions: list[dict]: Eine Liste von Fragen, wobei jede Frage ein Dictionary ist, das mindestens einen Schlüssel type enthält.

    • Logik: Iteriert über die Liste der Fragen und ordnet die korrekte Antwort (correct oder accepted) dem Frage-ID zu, basierend auf dem type des Frage-Typs:

  • **"single"**: Die korrekte Antwort wird als einzelner Wert gespeichert.
  • **"multi"**: Die korrekte Antwort wird als Liste von Werten gespeichert.
  • **"text"**: Die akzeptierte Antwort wird als einzelner Wert gespeichert.

    • Rückgabe: Ein Dictionary, das die Frage-ID als Schlüssel und die korrekte Antwort (als Wert) als Wert enthält.

4. Zweck und Implikationen

Basierend auf den Importen und der Struktur der Funktionen lässt sich ableiten, dass test_curriculum.py primär folgende Aufgaben erfüllt:

  1. Test-Setup/Helper-Funktionen: Die Hilfsfunktionen (_correct_answers) deuten darauf hin, dass die Tests oder die Hauptlogik die korrekten Lösungen für Fragen verarbeiten müssen.
  2. Datenabfrage: Die Importe von get_course und list_courses zeigen, dass die Tests oder die Hauptlogik Kursdaten aus dem Katalog abfragen.
  3. Datenpersistenz: Die Verwendung von SchoolRepository impliziert, dass die Tests oder die Anwendung die Fähigkeit besitzt, Daten in die Datenbank zu schreiben oder zu lesen.
  4. Sicherheit/Authentifizierung: Die _Session-Klasse deutet darauf hin, dass die Tests oder die Anwendung Benutzerkontexte (Login, Rollen wie Lehrer) verwalten müssen.
  5. Dateiverarbeitung: Die Importe von zipfile und tempfile deuten darauf hin, dass Tests möglicherweise Dateien (z.B. Kursmaterialien) temporär erstellen, komprimieren oder verwalten müssen.

5. Zusammenfassung der Struktur

Die Datei ist modular aufgebaut und trennt die Datenverarbeitung (Antwortkorrektur), die Kontextverwaltung (Session) und die Schnittstellen zu den Kernsystemkomponenten (CurriculumService, Repository, Catalog) voneinander. Sie dient als Basis für die Validierung der Funktionalität des Lehrplansystems.

Technische Dokumentation: Python Curriculum Payload Generator

1. Übersicht

Dieses Dokument beschreibt die Funktion _custom_course_payload aus der Datei test_curriculum.py. Diese Funktion dient dazu, ein strukturiertes JSON-ähnliches Datenobjekt (Payload) für einen benutzerdefinierten Kurs zu generieren. Das generierte Payload enthält detaillierte Informationen über den Kursinhalt, Zielgruppe, Bewertungsparameter und spezifische Module sowie eine abschließende Prüfung.

2. Funktion

Funktion: _custom_course_payload() -> dict Beschreibung: Diese interne Funktion konstruiert und gibt ein komplettes Dictionary zurück, das alle notwendigen Metadaten und Inhalte für einen spezifischen Python-Kurs enthält. Parameter: Keine Rückgabe: Ein Python-Dictionary (dict), das die Kursstruktur darstellt.

3. Struktur des generierten Payloads

Das zurückgegebene Dictionary folgt einer hierarchischen Struktur, die in Hauptkomponenten unterteilt ist:

3.1. Kurs-Metadaten (Top-Level)

Diese Schlüssel definieren die grundlegenden Informationen zum Kurs:

  • course_id: eindeutige Kennung des Kurses ("python-werkstatt-eigenkurs").
  • title: Der Haupttitel des Kurses ("Python Werkstatt").
  • subtitle: Eine ergänzende Beschreibung ("Eigener schulischer Aufbaukurs").
  • subject_area: Das Fachgebiet ("Python Projekte").
  • summary: Eine kurze Zusammenfassung des Kurses ("Lehrkraft-eigener Modullehrplan fuer Projektarbeit.").
  • audience: Die Zielgruppe ("Klasse 9/10").
  • estimated_hours: Geschätzte Dauer in Stunden (4).
  • certificate_title: Der Titel des zu erstellenden Zertifikats ("Nova School Zertifikat Python Werkstatt").
  • pass_ratio: Erfolgsquote für die Module (0.6).
  • final_pass_ratio: Erfolgsquote für die Abschlussprüfung (1.0).
  • certificate_theme: Ein verschachteltes Dictionary, das die visuellen/thematischen Einstellungen für das Zertifikat definiert.
  • **certificate_theme Struktur:**
  • label: Der Hauptname des Themas ("Python Werkstatt").
  • accent: Primäre Akzentfarbe (Hex-Code: #126d67).
  • accent_dark: Dunklere Akzentfarbe (Hex-Code: #0a4d49).
  • warm: Eine warme Farbe (Hex-Code: #8f412f).
  • paper: Die Hintergrundfarbe für das Zertifikatspapier (Hex-Code: #fbf3e5).

3.2. Module (modules)

Ein Array von Objekten, das die einzelnen Lernmodule des Kurses definiert. Struktur eines Module-Objekts:

  • module_id: Eindeutige ID des Moduls ("m01_funktionen").
  • title: Titel des Moduls ("Funktionen in Projekten").
  • estimated_minutes: Geschätzte Dauer des Moduls in Minuten (35).
  • objectives: Ein Array von spezifischen Lernzielen für das Modul (z.B. ["Funktionen planen", "Parameter einsetzen"]).
  • lesson_markdown: Der Inhalt des Moduls im Markdown-Format, der die Lehrmaterialien enthält.
  • quiz_pass_ratio: Erfolgsquote für die Module-Quiz (0.5).
  • questions: Ein Array von Quizfragen innerhalb dieses Moduls.

    • **Struktur einer Frage (question):**

  • id: Eindeutige Frage-ID (z.B. "f1").
  • type: Art der Frage ("single" für Multiple Choice, "text" für Freitext).
  • prompt: Die eigentliche Frage.
  • options (Nur bei type="single"): Array von Optionen, wobei jede Option ein id, ein label und eine correct-Liste enthält.
  • correct: Eine Liste von Option-IDs, die die korrekte Antwort darstellen.
  • points: Die Punktzahl für diese Frage (1).
  • explanation: Eine detaillierte Erklärung zur korrekten Antwort.
  • placeholder (Nur bei type="text"): Ein Platzhaltertext für die Benutzerantwort.

3.3. Abschlussprüfung (final_assessment)

Ein Objekt, das die Details der abschließenden Prüfung speichert. Struktur der Abschlussprüfung:

  • assessment_id: Eindeutige ID der Prüfung ("python-werkstatt-final").
  • title: Titel der Abschlussprüfung ("Abschlusspruefung Python Werkstatt").
  • instructions: Allgemeine Anweisungen für die Prüfung.
  • questions: Ein Array von Fragen für die Abschlussprüfung. (Die Struktur der Fragen ist identisch mit der Struktur der Fragen innerhalb eines Moduls, wobei die type und die prompt spezifisch für die finale Prüfung sind).

4. Anwendungsfälle

Dieses Payload ist primär für Systeme gedacht, die:

  1. Lernmanagement-Plattformen (LMS): Zur Erstellung von Kursstrukturen und Inhalten.
  2. Zertifikatsgeneratoren: Zur Definition der Inhalte, die auf Zertifikate übertragen werden sollen.
  3. Content-Management-Systeme (CMS): Zur Speicherung von Kursmetadaten und Lernmaterialien.

5. Technische Hinweise

  • Datenformat: Das generierte Objekt ist ein Python-Dictionary, das strukturell JSON-kompatibel ist.
  • Datenintegrität: Die Struktur stellt sicher, dass alle notwendigen Komponenten (Metadaten, Module, finale Prüfung) vorhanden sind, um einen vollständigen Kurs zu definieren.
  • Flexibilität: Die Verwendung von type-Feldern (z.B. single, text) ermöglicht eine flexible Definition verschiedener Fragenformate innerhalb des Payloads.

Technische Dokumentation: _bundle_archive_bytes Funktion

1. Übersicht

Die Funktion _bundle_archive_bytes ist eine interne Funktion, die ein strukturiertes Datenpaket (ein Archiv in ZIP-Format) basierend auf bereitgestellten Parametern erstellt. Dieses Paket dient wahrscheinlich dazu, einen vollständigen Lernmodul oder einen Kursbaustein für eine bestimmte Python-Lernsequenz zu verpacken, inklusive Metadaten, Kursinhalten, Materialvorlagen und Mentor-Regeln. Die Funktion generiert einen ZIP-Archiv-Bytes-Stream, der die folgenden Komponenten enthält:

  1. manifest.json: Metadaten über das gesamte Bundle.
  2. signature.json: Eine kryptografische Signatur des Manifests.
  3. courses/python-grundlagen.json: Die detaillierten Kursdaten.
  4. material_presets/python-example.json: Die Materialvorlage für ein spezifisches Beispiel.
  5. mentor_rules/python-einstieg.json: Die Anweisungen für den Mentor/Lehrer.

2. Funktion

def _bundle_archive_bytes(*, secret: str, version: str = "2026.03", module_title: str = "Einfuhrung in Python und Programmieren") -> bytes:
# ... (Implementierung)

2.1. Parameter

Parameter Typ Pflicht? Beschreibung Standardwert
secret str Ja Ein geheimes Schlüsselwort (Secret Key) zur kryptografischen Signaturung. Nicht angegeben
version str Nein Die Versionsnummer des Bundles. Wird verwendet, um den bundle_id zu generieren. "2026.03"
module_title str Nein Der Titel des spezifischen Moduls, das verpackt wird. "Einfuhrung in Python und Programmieren"

2.2. Rückgabe

3. Funktionsweise und Logik

Die Funktion folgt einem mehrstufigen Prozess:

3.1. Generierung der Identifikatoren und Datenstrukturen

  1. **bundle_id Generierung:**

    2. Ein eindeutiger ID für das Bundle wird erstellt, basierend auf der version (z.B. support-python-2026-03).

  2. **course_payload (Kursdaten):**

    3. Ein komplexes Dictionary, das alle Informationen zum Kurs enthält, einschließlich:

  • Allgemeine Kursdetails (course_id, title, audience, estimated_hours).
  • Zertifikatsparameter (certificate_title, pass_ratio).
  • **modules:** Eine Liste von Lernmodulen. Das erste und einzige definierte Modul (p01_einstieg) enthält spezifische Details wie Lernziele, Zeitangaben, Quiz-Informationen und Fragen.
  • **final_assessment:** Details zur Abschlussprüfung.
  1. **material_preset (Materialvorlage):**

    2. Ein Dictionary, das eine Vorlage für Code-Beispiele bereitstellt, das spezifisch für das erste Modul (p01_einstieg) konfiguriert ist.

  2. **mentor_rule (Mentor-Regeln):**

    3. Ein Dictionary, das Anweisungen für den Lernbegleiter enthält, wie er mit den Lernenden interagieren soll (z.B. welche Fragen gestellt werden sollen, welche Themen vermieden werden).

  3. **manifest (Manifest):**

    4. Ein Dictionary, das die grundlegenden Informationen über das generierte Archiv enthält (ID, Schema-Version, Version, Ersteller, Erstellungsdatum).

3.2. Erstellung des Kanonischen JSON-Strings

Das gesamte generierte Datenpaket (Manifest, Courses, Presets, Rules) wird in einen JSON-String serialisiert.

  • Formatierung: json.dumps wird verwendet, um das JSON zu erstellen.
  • Sortierung: sort_keys=True stellt sicher, dass die Schlüssel im JSON-String alphabetisch sortiert sind, was für konsistente Archivierung wichtig ist.
  • Zeichenkodierung: ensure_ascii=False stellt sicher, dass nicht-ASCII-Zeichen (wie Akzente in deutschen Titeln) korrekt dargestellt werden.
  • Trennung: separators=(",", ":") entfernt unnötige Leerzeichen zwischen Schlüssel-Wert-Paaren, was die Dateigröße optimiert.

3.3. Erstellung der Signatur

  1. Kanonischer String: Der generierte JSON-String wird in Bytes umgewandelt und für die kryptografische Operation vorbereitet.
  2. SHA-256-Hash: Der kanonische String wird mittels hashlib.sha256 hasht.
  3. HMAC-Signatur: Eine Hash-Minisum-Authentifizierung (HMAC) wird durchgeführt:
  • Algorithmus: hmac-sha256.
  • Schlüssel: Der bereitgestellte secret.
  • Daten: Der kanonische JSON-String.
  • Ergebnis: Die canonical_sha256 (für Verifizierung) und die signature (für Authentifizierung) werden generiert.

3.4. Archivierung (ZIP-Format)

  1. Buffer-Setup: Ein io.BytesIO-Objekt wird erstellt, um den gesamten Archivinhalt im Speicher zu sammeln.
  2. ZIP-Erstellung: Ein zipfile.ZipFile wird mit dem BytesIO-Buffer geöffnet.
  3. Inhaltsschreibvorgang: Die folgenden fünf Dateien werden in das Archiv geschrieben:
  • manifest.json
  • signature.json
  • courses/python-grundlagen.json
  • material_presets/python-example.json
  • mentor_rules/python-einstieg.json
  1. Rückgabe: Die gesamten Bytes des erstellten ZIP-Archivs (buffer.getvalue()) werden zurückgegeben.

4. Abhängigkeiten

Die Funktion setzt folgende Standard-Bibliotheken voraus:

  • json: Für die Serialisierung und Deserialisierung von JSON-Daten.
  • time: Für die Erstellung des Zeitstempels.
  • hashlib: Für die Berechnung der SHA-256-Hashes.
  • hmac: Für die Berechnung der HMAC-Signatur.
  • io: Für die Verwendung von BytesIO als in-memory-Stream.
  • zipfile: Für die Erstellung des ZIP-Archivs.

5. Verwendung und Anwendungsfall

Diese Funktion wird verwendet, um komplexe, mehrteilige Lernpakete (z.B. für einen Kursbaustein) in ein standardisiertes, sicheres und verifizierbares Archiv zu verpacken. Anwendungsfall: Ein System, das Lernmaterialien dynamisch generiert und diese dann als vollständige, signierte Pakete an Benutzer oder Lernplattformen bereitstellt.

6. Sicherheitshinweise

  • Vertraulichkeit: Der secret-Parameter ist kritisch. Er muss streng vertraulich behandelt werden, da er direkt zur kryptografischen Signatur des gesamten Pakets verwendet wird.
  • Authentizität: Die signature.json ermöglicht es dem Empfänger, die Integrität und die Herkunft des Pakets zu überprüfen, indem er die Daten mit dem secret neu signiert und den Hash vergleicht.
  • Datenintegrität: Die Verwendung von JSON-Serialisierung mit sortierten Schlüsseln stellt sicher, dass die Struktur der Daten konsistent ist, was die Verifizierung erleichtert.
