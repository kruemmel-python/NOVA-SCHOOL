Technische Dokumentation: pty_host.py

Version: 1.0 Datum: 2023-10-27 Autor: [Ihr Name/Team] Beschreibung: Diese Datei implementiert die Logik für einen Host-Prozess, der wahrscheinlich eine Pseudo-Terminal-Umgebung (PTY) verwaltet oder eine Verbindung zu einem Terminal-Prozess herstellt. Sie beinhaltet Funktionen zur Normalisierung von Terminalgrößen und die notwendigen Importe für Prozesssteuerung und Multithreading.

1. Überblick

Die Datei pty_host.py dient als Basis für einen Prozess, der die Schnittstelle zwischen einem Benutzer und einem externen Prozess (wahrscheinlich über eine PTY) verwaltet. Die Implementierung umfasst Funktionen zur Anpassung der Terminaldimensionen und die notwendigen Bibliotheken für die Interaktion mit dem Betriebssystem (Prozess- und Signalmanagement).

2. Funktionen

2.1. def normalize_terminal_size(cols: int | None, rows: int | None) -> tuple[int, int]

Beschreibung: Diese Funktion nimmt optional die gewünschte oder aktuelle Terminalbreite (cols) und Höhe (rows) entgegen und normalisiert diese Werte auf einen definierten, sicheren Bereich. Parameter:

  • cols (int | None): Die gewünschte oder aktuelle Terminalbreite. Wenn None, wird ein Standardwert verwendet.
  • rows (int | None): Die gewünschte oder aktuelle Terminalhöhe. Wenn None, wird ein Standardwert verwendet.

    • Rückgabe:

  • tuple[int, int]: Ein Tupel, das die normalisierten Breite und Höhe zurückgibt.

    • Logik: Die Funktion setzt harte Grenzen (Constraints) für die Terminaldimensionen, um unerwartete oder zu große/kleine Dimensionen zu verhindern:

  • **Breite (width):** Wird auf einen Bereich zwischen 40 und 320 (inklusive der Eingabe, falls vorhanden) begrenzt.
  • **Höhe (height):** Wird auf einen Bereich zwischen 10 und 120 (inklusive der Eingabe, falls vorhanden) begrenzt.

    • Beispiel:

# Beispielaufruf
width, height = normalize_terminal_size(cols=500, rows=50)
# width wird 320, height wird 120

Technische Dokumentation: create_pty_process Funktion

1. Funktion

Name: create_pty_process Beschreibung: Diese Funktion ist verantwortlich für die Erstellen eines Prozesses, der eine Pseudo-Terminal-Schnittstelle (PTY) simuliert. Sie nimmt einen Befehl, einen Arbeitsverzeichnispfad, eine Umgebungsvariablen-Satz und optional Terminal-Dimensionen entgegen, um den entsprechenden Prozess-Erstellungsmechanismus (Windows oder POSIX) zu wählen und zu initialisieren. Modul: (Annahme: Dieses Modul gehört zu einer Bibliothek, die Terminal- oder Prozessmanagement-Funktionalität bereitstellt.)

2. Signatur

def create_pty_process(command: Sequence[str], cwd: Path, env: dict[str, str], cols: int | None = None, rows: int | None = None) -> PtyProcess:

3. Parameter

Parameter Typ Beschreibung Pflicht?
command Sequence[str] Der Befehl, der ausgeführt werden soll (z.B. ein Shell-Befehl). Ja
cwd Path Der Arbeitsverzeichnispfad, in dem der Prozess gestartet werden soll. Ja
env dict[str, str] Ein Dictionary von Umgebungsvariablen, die dem Prozess zur Verfügung gestellt werden sollen. Ja
cols `int None` Optionale Anzahl der Spalten (Breite) für die Terminal-Dimensionierung. Wenn None, wird die Terminalgröße automatisch normalisiert. Nein
rows `int None` Optionale Anzahl der Zeilen (Höhe) für die Terminal-Dimensionierung. Wenn None, wird die Terminalgröße automatisch normalisiert. Nein

4. Rückgabe

  • Typ: PtyProcess
  • Beschreibung: Ein Objekt vom Typ PtyProcess, das den erstellten und initialisierten Pseudo-Terminal-Prozess repräsentiert.

5. Logik und Verhalten

Die Funktion folgt einer Plattform-abhängigen Logik, um die korrekte Implementierung für das Erstellen des PTY-Prozesses zu wählen:

  1. Terminal-Dimensionen Normalisierung:
  • Zuerst werden die Eingaben für cols und rows verwendet, um die tatsächliche Breite (width) und Höhe (height) zu berechnen, mithilfe der Funktion normalize_terminal_size(cols, rows).
  1. Plattformprüfung (OS-Abhängigkeit):
  • **Windows (os.name == "nt"):**
  • **WinptyProcess vorhanden:** Wenn die Klasse WinptyProcess definiert ist, wird diese Klasse verwendet, um den Prozess zu erstellen.
  • Andernfalls: Es wird die Klasse _WindowsConPtyProcess verwendet.
  • POSIX-Systeme (Linux, macOS etc.):
  • Die Klasse _PosixPtyProcess wird verwendet, um den Prozess zu erstellen.
  1. Prozessinitialisierung:
  • Basierend auf der Plattform wird die entsprechende Unterklasse (_PyWinPtyProcess, _WindowsConPtyProcess, oder _PosixPtyProcess) aufgerufen, um den eigentlichen PTY-Prozess zu instanziieren und zu starten.

6. Abhängigkeiten

  • os (für die Erkennung des Betriebssystems)
  • Path (von pathlib oder ähnlichem, für Pfadmanipulation)
  • PtyProcess (Klasse, die das Ergebnis der Funktion darstellt)
  • normalize_terminal_size (Externe Hilfsfunktion zur Berechnung der Terminalgröße)
  • WinptyProcess (Klasse, spezifisch für Windows-Implementierung)
  • _WindowsConPtyProcess (Klasse, spezifisch für Windows-Implementierung)
  • _PosixPtyProcess (Klasse, spezifisch für POSIX-Implementierung)

7. Anwendungsfälle

Diese Funktion wird verwendet, wenn eine Anwendung benötigt, um einen externen Befehl in einer Umgebung zu starten, die eine interaktive Terminal-Schnittstelle (PTY) simuliert, sei es für die Interaktion mit einem Remote-Server oder zur Simulation einer lokalen Terminal-Umgebung.
